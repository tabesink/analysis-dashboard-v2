# DMD-20-02 — Open metadata editing from a Database version-row pencil

Type: AFK

Blocked by: DMD-20-01

User stories covered: 1, 2, 5

## What to build

Replace the visible version-row event-count text in the Database table with a compact pencil action. When clicked, the action opens a settings-style modal dialog preselected to that version's program/version scope. The dialog uses a left-side navigation rail with one item, **Edit Metadata**, and renders the reusable Edit Metadata panel in the right-side content area.

Opening and closing the dialog must keep the user anchored in the Database table: no route change, no lost filters, no lost expanded rows, and no selection changes.

## Acceptance criteria

- [ ] Version group rows show a compact pencil action where the event-count text used to appear.
- [ ] The pencil has a descriptive accessible label that includes the target program/version and, if useful, the event count.
- [ ] Clicking the pencil opens the metadata edit dialog for the correct program/version.
- [ ] Clicking the pencil does not toggle row expansion and does not change checkbox selection.
- [ ] The dialog matches the settings-style structure: centered constrained modal, subtle overlay, left navigation rail, right content panel, and close control.
- [ ] The initial left navigation contains only **Edit Metadata**.
- [ ] The right panel renders the scoped Edit Metadata panel with fields enabled or disabled according to the current user permissions.
- [ ] Closing the dialog returns to the same Database table state without navigation.
- [ ] Component tests cover the pencil trigger, event propagation guard, correct scope passed to the dialog, and basic dialog rendering.
