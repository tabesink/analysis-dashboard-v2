# DPR-31-04 — Inspect API is strict read model

Type: AFK

Blocked by: DPR-31-03

User stories covered: 2, 3, 4

## Context packet

- `../PRD.md`
- `../HANDOFF.md`
- `../IMPLEMENTATION_MAP.md`
- `../ARCHITECTURE_SKETCH.md`

## Previous slice provides

- Simplified persisted task/cell states from schedule-triggered calculations.

## This slice changes

Refactor Inspect Damage backend endpoints/services so inspect is read-only:

- never starts calculation
- never triggers backfill
- never mutates stale/repair state on read

The inspect response must always include selected event rows, even when no damage cells exist yet, and may include read-only task status metadata for running/failed banners.

## This slice must not rework

- Schedule command trigger behavior.
- Frontend rendering implementation details beyond API contract updates.

## Next slice can assume

- Inspect endpoints are safe query-only surfaces.
- Selected rows can render with empty/unavailable cells before calculation completes.

## Acceptance criteria

- [x] Inspect request path starts no new task and calls no mutation/backfill service.
- [x] Inspect response includes selected event rows even with zero persisted damage rows.
- [x] Inspect response surfaces running/failed task context as read-only metadata when relevant.
- [x] Backend tests prove inspect is query-only and non-mutating.
