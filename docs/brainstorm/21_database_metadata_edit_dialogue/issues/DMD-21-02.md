# DMD-21-02 — Add Assign Channels to the Database metadata dialog

Type: AFK

Blocked by: DMD-21-01

User stories covered: 1, 2, 6

## What to build

Extend `MetadataEditDialog` so the left navigation rail includes **Assign Channels** alongside **Edit Metadata**. The dialog still opens from the version-row pencil, still defaults to **Edit Metadata**, and still preselects the clicked program/version. Selecting **Assign Channels** renders the reusable Assign Channels panel in the right content area with the same scoped program/version.

Opening, switching sections, and closing the dialog must keep the user anchored in the Database table: no route change, no lost filters, no lost expanded rows, and no selection changes.

## Acceptance criteria

- [ ] The dialog left nav shows **Edit Metadata** and **Assign Channels**.
- [ ] Opening from the pencil defaults to **Edit Metadata**.
- [ ] Clicking **Assign Channels** shows the scoped Assign Channels panel for the dialog's program/version.
- [ ] Switching sections does not change program/version scope and does not navigate away from the Database table.
- [ ] Both sections retain in-memory draft state while the dialog remains open.
- [ ] The dialog header still shows the active program/version context in both sections.
- [ ] Nav items have descriptive accessible labels and an active-state indicator.
- [ ] Component tests cover nav rendering, default section, section switching, and scoped panel mounting.
