## Previous slice provides

- Raw Reference/Target event x channel facts filtered by selected event IDs, selected channel keys, and damage-cell status from `DamageComparisonState`.

## This slice changes

Add a pure comparison aggregation module that produces the v1 plot data families from raw facts: cumulative damage by program/version, absolute damage by event, cumulative damage by channel, and Target-vs-Reference channel deltas.

The module must support `DamageComparisonState.value_mode`, signed deltas, percent difference, ratio, normalized ratio, and low-reference guard semantics.

## This slice must not rework

- UI chart components except minimal test fixtures if needed.
- Session persistence.
- Event-tree or channel-picker UI.
- Backend aggregation APIs.

## Next slice can assume

- Aggregated plot series are available from pure functions.
- Absolute and normalized outputs are explicit in returned metadata.
- Low-reference rows are flagged without misleading finite percent/ratio values.
- Plot UI can consume consistent structures for all v1 plot families.

## Acceptance criteria

- [ ] Cumulative program/version damage groups by dataset, program ID, version, and selected channels.
- [ ] Event damage groups by dataset, event ID, and channel and can support stacked/grouped channel bars.
- [ ] Cumulative channel damage groups by dataset and channel.
- [ ] Target-vs-Reference channel comparison computes absolute delta, percent difference, ratio, and normalized ratio.
- [ ] Absolute/normalized output is selected from `comparison.value_mode`.
- [ ] Normalized values follow the PRD formulas and include explicit mode/denominator metadata for labels.
- [ ] Low-reference rows are flagged when reference damage is missing, zero, or below threshold, and ratio-style outputs are not falsely finite.
- [ ] Unit tests cover each aggregation family, normalized outputs, signed deltas, and low-reference guard behavior.
