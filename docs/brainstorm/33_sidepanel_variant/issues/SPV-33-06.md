## Previous slice provides

- Reference and Target side-panel sections wired to `DamageComparisonState`.
- Plotted-channel selection wired to `comparison.selected_channel_keys`.

- Pure aggregation outputs for all v1 plot families.

## This slice changes

Wire the complete comparison route workflow around the existing comparison session helpers: side-panel selections, value-mode updates, union inspect-damage data fetching, raw fact building, aggregation, plot rendering, empty states, legends/subtitles, regression tests, and docs closeout.

The resulting workflow should be demoable end-to-end: choose Reference events, choose Target events, choose plotted channels, switch absolute/normalized mode, and see v1 comparison plots update from existing inspect-damage data.

## This slice must not rework

- Fatigue damage calculation.
- Database upload, certificate, edit-metadata, or unrelated side panels.
- Backend pre-aggregation unless client-side union inspect data is proven insufficient.
- `all_in_program_version` behavior beyond preserving it as an explicit non-default state value unless product has already requested a UI.

## Acceptance criteria

- [ ] Comparison route renders Filter Data, Reference Load Data, Target Load Data, plotted-channel controls, and absolute/normalized mode controls.
- [ ] Reference, Target, channel, and value-mode controls update comparison state through the existing session merge path without replacing table preferences.
- [ ] Plot data requests use the union of Reference and Target selected event IDs and reuse existing inspect-damage results.
- [ ] Plots update after Reference, Target, channel, or value-mode changes.
- [ ] V1 plot families render: cumulative program/version damage, event damage, cumulative channel damage, and Target-vs-Reference channel deltas.
- [ ] Plot legends and subtitles state absolute/normalized mode, selected event counts, selected channel count, and relevant low-reference or missing-data context.
- [ ] Empty states guide the user when Reference events, Target events, selected channels, or usable damage values are missing.
- [ ] Existing dashboard Load Data and inspect-damage table/plot preferences do not regress.
- [ ] `CHANGELOG.md` is updated for the user-facing comparison workflow.
- [ ] `../HANDOFF.md` and `../IMPLEMENTATION_MAP.md` are updated with completion notes or changed assumptions.
