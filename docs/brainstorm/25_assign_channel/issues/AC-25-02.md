# AC-25-02 — Modal-only feedback for Assign Channels save and upload

Type: AFK

Blocked by: AC-25-01

User stories covered: 1, 2, 3, 4, 5, 6, 7, 8, 15, 16, 17, 18, 19, 21, 26, 27, 28, 30

## What to build

Remove transitional loading toasts from Assign Channels save and channel-map YAML upload success paths. The derived-data progress modal becomes the sole in-flight feedback channel after the start API returns a `channel_reprocess` task id.

Keep error toasts for validation failures and start API errors. Keep unrelated success toasts for reset and pre-reset restore. Ensure save and upload still call task tracking immediately after start response and that active-task reuse reopens the existing modal without duplicate polls.

## Acceptance criteria

- [ ] Manual Assign Channels save does not show a loading toast that dismisses before reprocess completes.
- [ ] Channel-map YAML upload does not show a loading toast on successful task start.
- [ ] Save/upload open or reopen the progress modal immediately after task id is received.
- [ ] Modal shows validating, extracting, and generating phases with live server messages.
- [ ] Modal shows elapsed time, progress bar, and close-without-cancel copy.
- [ ] Completion summary shows success, partial failure, or hard failure using existing completion builder.
- [ ] Successful completion invalidates the same queries as current channel-map save completion.
- [ ] Starting save/upload during an active task reopens the existing task.
- [ ] Edit Metadata inline banner still reopens progress after modal dismiss.
- [ ] Client tests cover save, upload, no-toast success paths, polling updates, dismiss/reopen, active-task reuse, and completion summaries.
