# AC-25-03 — Database-page banner for background channel reprocess

Type: AFK

Blocked by: AC-25-01

User stories covered: 11, 12, 13, 14, 20, 27

## What to build

When channel reprocess is running, the progress modal is dismissed, and Edit Metadata is closed, show a compact banner on the Database page with program/version scope and a Reopen progress action. This prevents background work from becoming invisible when users return to the event tree after closing the metadata editor.

Reuse channel reprocess store state. Do not introduce a global app-shell task center.

## Acceptance criteria

- [ ] Banner appears on Database page when a channel reprocess task is running, modal is dismissed, and Edit Metadata is not open for that scope.
- [ ] Banner shows program id and version for the active scope.
- [ ] Banner shows the current live progress message or a short running label.
- [ ] Reopen progress opens the shell-mounted modal for that scope.
- [ ] Banner does not appear after completion once summary is acknowledged and scoped store entry is cleared.
- [ ] Banner does not duplicate the Edit Metadata inline banner when the editor is open for the same scope.
- [ ] Read-only users can see running state without write actions on the banner.
- [ ] Client tests cover banner visibility, reopen action, and editor-open suppression.
