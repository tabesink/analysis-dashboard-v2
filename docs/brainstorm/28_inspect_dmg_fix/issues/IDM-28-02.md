# IDM-28-02 — Persist index-based channel map

Type: AFK

Blocked by: IDM-28-01

User stories covered: 3, 7, 8, 21, 22, 36

## What to build

Change Assign Channels save and channel-map YAML upload so the version-wide channel map persists **index-based lookup names** (`col_N`) rather than title strings resolved from the first retained artifact preview.

Editor preview may still show human-readable titles from a preview artifact for UX, but durable storage must not freeze one event’s export titles across the program/version. Channel map snapshots should remain consistent with normalized index-based plot definitions.

## Acceptance criteria

- [x] After UI save, persisted channel map rows use `col_N` for `x_channel` / `y_channel` lookup fields.
- [x] YAML upload path matches UI save persistence semantics.
- [x] Snapshot normalization continues to store plot indices and generic names.
- [x] Server tests assert saved map is index-based even when preview artifact uses full Moog titles.
- [x] No regression to column-count validation or plot-order persistence.
