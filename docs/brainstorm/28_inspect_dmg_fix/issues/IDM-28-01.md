# IDM-28-01 — Shared per-event channel resolver

Type: AFK

Blocked by: None

User stories covered: 17, 18, 19, 20, 24, 37

## What to build

Extract a shared **per-event channel resolution** module used by derived-data readers:

- Pure resolver: given plot mapping (`x_col`, `y_col`) + header list (+ optional units), return resolved channel names or structured errors (`missing_headers`, `column_out_of_range`).
- Header provider adapter: load headers/units for an `event_id` from event preview metadata first, then ingestion artifact preview `#TITLES` fallback.

Move logic equivalent to today’s “apply column indices to header row” behavior out of ingestion-only helpers so damage and reprocess can share it. Keep legacy generic `col_N` pattern matching as a separate fallback helper, not the primary path.

## Acceptance criteria

- [ ] Pure resolver unit tests cover valid indices, out-of-range indices, and empty headers.
- [ ] Header provider tests cover event preview path and artifact preview fallback.
- [ ] Resolver returns abbreviated and full Moog-style titles correctly from respective fixture header rows at the same indices.
- [ ] No database writes in the pure resolver core.
- [ ] Public interface is small enough for damage and ingestion to depend on without importing ingestion service internals.
