# UPF-29-04 — Add guarded legacy channel-map repair

Type: AFK

Blocked by: UPF-29-03

User stories covered: Architecture review finding C1

## What to build

Provide a guarded repair path for program/versions that still have legacy channel maps persisted with first-file human-readable title strings. The repair should normalize the saved map to the index-based `col_N` contract when column indices are available, then reuse the existing channel reprocess and damage calculation flows to regenerate derived data.

The repair must be explicit, scoped to one program/version, and safe to run repeatedly. It should not silently mutate data on page load.

## Acceptance criteria

- [ ] The repair path detects whether a program/version has a legacy explicit-title map that can be normalized from stored column indices.
- [ ] Write users can trigger repair only for a scoped program/version they are allowed to modify.
- [ ] The repair is idempotent: running it again on an already normalized map does not create duplicate work or corrupt mappings.
- [ ] Repair starts or reuses the existing channel reprocess task and lets the existing post-reprocess precompute path handle damage calculation when prerequisites are ready.
- [ ] If repair cannot proceed, the response explains why without partially mutating the map.
- [ ] Server tests cover normalizable legacy map, already-normalized map, blocked/unauthorized access, and active-task reuse.
- [ ] Client tests cover invoking repair from the guidance added in `UPF-29-03`.
- [ ] No filename heuristics or expanded title-synonym tables are introduced.
