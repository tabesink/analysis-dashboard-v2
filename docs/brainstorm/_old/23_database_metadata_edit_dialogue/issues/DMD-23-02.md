# DMD-23-02 — Add Durability Schedule to the metadata popup dialog

Type: AFK

Blocked by: DMD-23-01

User stories covered: 1, 2, 3, 4, 8, 9, 10, 17, 20, 21

## What to build

Extend the Database table pencil popup dialog so its left navigation rail includes **Durability Schedule** directly below **Assign Channels**. The dialog should still open on **Edit Metadata** by default, remain scoped to the clicked program/version, and render the reusable Durability Schedule panel in the right content area when the new route is selected.

Opening the dialog, switching to Durability Schedule, uploading or reviewing a schedule, and closing the dialog must keep the user anchored in the Database table without route navigation, table filter loss, expanded-row loss, scroll loss, or selection changes.

## Acceptance criteria

- [ ] The dialog left nav shows **Edit Metadata**, **Assign Channels**, and **Durability Schedule** in that order.
- [ ] **Durability Schedule** appears directly below **Assign Channels**.
- [ ] Opening the dialog from the pencil still defaults to **Edit Metadata**.
- [ ] Selecting **Durability Schedule** shows the scoped Durability Schedule panel for the dialog's program/version.
- [ ] Switching sections does not change program/version scope and does not navigate away from the Database table.
- [ ] The dialog header still shows the active Job ID and Version while Durability Schedule is active.
- [ ] The Durability Schedule section exposes the active schedule summary, no-schedule state, upload affordance, schedule table, Reset, and Save through the popup.
- [ ] The new nav item has a descriptive accessible label and active-state indicator.
- [ ] Component tests cover nav rendering, default section, ordering, scoped panel mounting, and visible Durability Schedule content.
