# PPU-27-01 — Auto-start damage after channel reprocess completion

Type: AFK

Blocked by: None

User stories covered: 1, 2, 7, 8, 10, 11, 16, 17, 24, 26, 27, 28, 29, 30, 33, 34, 37, 39, 40

## What to build

Add the first post-upload precompute path: when `channel_reprocess` completes for a program/version, evaluate whether an active durability schedule exists and whether damage prerequisites are current. If they are ready, start or reuse the existing `damage_calculation` task for that program/version. If they are not ready, do not create a doomed task and return a blocked/no-op decision that can be surfaced by existing client feedback paths.

This slice should introduce the small deterministic precompute orchestrator behind the channel-completion workflow. The orchestrator should expose a simple decision result such as no-op, start/reuse damage calculation, or blocked, while keeping the public task kinds limited to `channel_reprocess` and `damage_calculation`.

## Acceptance criteria

- [x] Completing a `channel_reprocess` task for a scheduled program/version starts a `damage_calculation` task when damage prerequisites are current.
- [x] Completing a `channel_reprocess` task reuses an already active derived-data task for the same program/version instead of starting a duplicate.
- [x] Completing a `channel_reprocess` task for an unscheduled program/version does not start damage calculation.
- [x] Completing a `channel_reprocess` task with missing or stale damage prerequisites does not create a damage task.
- [x] The orchestrator returns a deterministic decision for the same program/version state and active task state.
- [x] The public derived-data task kinds remain `channel_reprocess` and `damage_calculation`; no new public task kind is added.
- [x] Previous successful damage rows are marked or remain stale when channel-derived inputs changed, rather than being deleted.
- [x] Server tests cover active schedule ready, no active schedule, prerequisite failure, active-task reuse, no-op when current, and stale preservation after channel input changes.

## Blocked by

None - can start immediately
