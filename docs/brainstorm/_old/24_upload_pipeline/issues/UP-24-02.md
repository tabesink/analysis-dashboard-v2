# UP-24-02 — Show channel reprocess progress in Assign Channels

Type: AFK

Blocked by: UP-24-01

User stories covered: 1, 2, 3, 4, 5, 6, 8, 9, 10, 11, 12, 13, 14, 15, 44, 45

## What to build

Replace the toast-only Assign Channels reprocess feedback with a reusable processing modal and scoped inline banner that consume the `channel_reprocess` task from UP-24-01.

Manual Assign Channels save and channel-map YAML upload should both open the progress modal, poll the task, show the locked live messages, and present a completion summary. Closing the modal should not cancel processing. If processing continues after the modal is closed, the selected program/version's Edit Metadata dialog should show an inline banner with a Reopen progress action.

## Acceptance criteria

- [ ] Manual Assign Channels save opens a progress modal after receiving a `channel_reprocess` task id.
- [ ] Channel-map YAML upload opens the same progress modal after receiving a `channel_reprocess` task id.
- [ ] The modal polls the lean task status endpoint and maps status/phase/current event/progress message into user-visible progress.
- [ ] The progress bar uses coarse phase bands plus completed event counts; it does not require per-plot/per-channel counters.
- [ ] The modal shows the locked copy for validation, raw load-history extraction, and cross-plot generation messages.
- [ ] Closing the modal clearly communicates that processing continues in the background.
- [ ] Closing the modal does not attempt to cancel the server task.
- [ ] A scoped inline banner appears in the Edit Metadata dialog while channel reprocess continues for that program/version.
- [ ] The inline banner can reopen the current progress modal.
- [ ] Starting Assign Channels save/upload while a task is already active reopens the existing task rather than showing a duplicate operation.
- [ ] Successful completion invalidates/refetches the same downstream queries previously refreshed by channel-map save/upload.
- [ ] Partial failure completion shows a warning summary using processed and failed artifact counts.
- [ ] Client tests cover manual save, YAML upload, polling updates, close-only behavior, inline banner reopening, active-task reuse, and completion/failure summaries.
