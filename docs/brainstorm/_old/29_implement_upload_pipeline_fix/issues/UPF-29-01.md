# UPF-29-01 — Show persisted damage errors in Inspect Damage

Type: AFK

Blocked by: None

User stories covered: Architecture review finding H1

## What to build

Make Inspect Damage distinguish between damage that has not been calculated and damage that was calculated but failed. Persisted damage rows with status `error` should be visible in the table as an error state with concise repair context instead of rendering as empty cells.

This slice should preserve current display behavior for `current` and `stale` values, keep read-only users read-only, and avoid triggering new background work just because an error is displayed.

## Acceptance criteria

- [ ] Inspect Damage renders an explicit error indicator for persisted `error` damage cells.
- [ ] The error indicator exposes the persisted error message or a concise fallback message without expanding the table layout excessively.
- [ ] Empty cells remain reserved for genuinely missing/uncomputed damage, not persisted failures.
- [ ] Stale values continue to render with the existing stale treatment.
- [ ] Read-only users can see error context but cannot start repair mutations from this slice.
- [ ] Client tests cover `current`, `stale`, `error`, and missing cells.
- [ ] Existing Inspect Damage backfill/calculate behavior is unchanged except for the improved visibility.

## Blocked by

None - can start immediately
