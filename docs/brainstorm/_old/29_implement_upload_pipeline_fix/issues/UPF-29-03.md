# UPF-29-03 — Surface upload and legacy recovery guidance

Type: AFK

Blocked by: UPF-29-01

User stories covered: Architecture review findings C1, H4

## What to build

Give users a visible next step when the upload/derived-data pipeline cannot automatically converge. This includes two common confusing states:

- Upload completes with events pending channel assignment.
- Inspect Damage contains persisted channel lookup failures or mixed `current`/`error` damage that likely requires channel-map normalization, channel reprocess, and damage recalculation.

The slice should turn these states into clear, scoped UI guidance and actionable entry points using the existing Assign Channels, channel reprocess, and Calculate Damage flows. It should not perform automatic legacy-data mutation.

## Acceptance criteria

- [ ] Upload completion with pending channel map shows a completion summary or CTA explaining that Assign Channels is required before derived damage can be trusted.
- [ ] Inspect Damage or the database metadata workflow surfaces a recovery message when persisted damage errors indicate a likely channel-map lookup mismatch.
- [ ] The recovery message explains the sequence: save/normalize channel map, run channel reprocess, then run damage calculation.
- [ ] The guidance links or routes users to the existing scoped Assign Channels and Calculate Damage affordances where possible.
- [ ] Read-only users see explanation-only guidance without write actions.
- [ ] Tests cover pending-channel-map upload guidance and legacy damage-error recovery guidance.
- [ ] No new derived-data task kind or global task center is introduced.
