## Previous slice provides

- Hardened route-scoped comparison state and persistence contract built on the existing frontend helpers.
- Independent Reference and Target event ID storage.
- Safe session merge/prune helpers.

## This slice changes

Parameterize the existing side-panel Load Data event-tree behavior so the comparison panel can render separate `Reference Load Data` and `Target Load Data` sections from the same filtered event catalog.

Each section must read/write the existing `DamageComparisonState.reference.selected_event_ids` or `DamageComparisonState.target.selected_event_ids`. Each section must keep independent checked state, All/None actions, counts, empty-state guidance, and accessible labels while preserving existing single-pool Load Data routes.

## This slice must not rework

- Channel selection semantics.
- Plot data construction or aggregation.
- Global filter ownership.
- Existing non-comparison side panels except where needed to keep shared primitives compatible.

## Next slice can assume

- The comparison side panel can render Reference and Target Load Data sections.
- Selecting, selecting all, or clearing one dataset does not mutate the other dataset.
- Existing dashboard/inspect-damage Load Data sections still behave as before.

## Acceptance criteria

- [x] Reference and Target sections render with distinct accessible labels and count subtitles.
- [x] Each section uses the existing program -> version -> event hierarchy and current global filters.
- [x] Reference selection writes only to `comparison.reference.selected_event_ids`.
- [x] Target selection writes only to `comparison.target.selected_event_ids`.
- [x] Reference All/None actions affect only Reference selected event IDs.
- [x] Target All/None actions affect only Target selected event IDs.
- [x] Empty-state guidance identifies whether Reference, Target, filters, or unavailable events are blocking comparison.
- [x] The same event may be selected in both Reference and Target.
- [x] Existing single-pool Load Data behavior has focused regression coverage.
