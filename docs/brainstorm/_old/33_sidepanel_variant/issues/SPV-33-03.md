## Previous slice provides

- Persisted `selected_channel_keys` in the existing route-scoped comparison state.
- Pruning behavior for channel keys that no longer exist.

## This slice changes

Add a plotted-channel picker that controls damage-analysis channel selection separately from metadata/table column visibility.

The picker should present canonical damage channel keys and labels, persist selected keys to `comparison.selected_channel_keys`, and make it clear that metadata columns like Work Order or Job Number do not control plot meaning.

## This slice must not rework

- Generic table column visibility semantics.
- Damage math or plot aggregation.
- Event-tree selection behavior.
- Canonical damage channel definitions except where an existing source needs a small adapter.

## Next slice can assume

- Comparison state exposes a validated selected-channel set.
- Plot builders can filter by selected damage channel keys.
- UI distinguishes plotted damage channels from table layout preferences.

## Acceptance criteria

- [x] Channel picker lists canonical damage channel labels backed by stable channel keys.
- [x] Selecting or clearing plotted channels updates `comparison.selected_channel_keys` and persists across refresh/navigation.
- [x] Channel state updates use `mergeDamageComparisonState()` or the route's equivalent session merge path rather than replacing unrelated comparison fields.
- [x] Metadata/table columns are not stored as selected damage channels.
- [x] Hiding/showing table layout columns does not silently change selected plotted channels.
- [x] Missing channel keys are pruned when the canonical channel list changes.
- [x] Behavior is covered by focused state/helper and component tests.
