## Previous slice provides

- Persisted Reference and Target selected event IDs on `DamageComparisonState`.
- Persisted selected damage channel keys on `DamageComparisonState`.
- Validated channel-key pruning behavior.

## This slice changes

Build a pure raw comparison dataset module that turns the union inspect-damage response plus `DamageComparisonState` into event x channel facts for Reference and Target.

The builder must split rows by dataset membership, preserve overlap events in both datasets when selected in both, filter to selected event IDs and selected channel keys, and surface invalid/missing/stale damage-cell status according to view rules.

## This slice must not rework

- Fatigue damage calculation.
- Backend aggregation endpoints.
- Plot rendering.
- Event-tree UI behavior.

## Next slice can assume

- A deterministic event x channel fact list exists for Reference and Target.
- Each fact carries dataset membership, event ID, program ID, version, channel key, channel label, damage value, and value/status metadata.
- Overlap events are represented once per selected dataset membership.
- Invalid or missing damage cells are handled consistently before aggregation.

## Acceptance criteria

- [ ] Union inspect-damage data is split into Reference and Target facts based on `comparison.reference.selected_event_ids` and `comparison.target.selected_event_ids`.
- [ ] An event selected in both Reference and Target appears in both dataset memberships.
- [ ] Facts are filtered to `comparison.selected_channel_keys` only.
- [ ] Metadata columns never become plot facts.
- [ ] Missing, null, stale, or error damage values are dropped or marked according to explicit view rules.
- [ ] Event IDs, program IDs, versions, channel keys, and channel labels are preserved on facts.
- [ ] Unit tests cover overlap, selected-channel filtering, and invalid damage-cell behavior.
