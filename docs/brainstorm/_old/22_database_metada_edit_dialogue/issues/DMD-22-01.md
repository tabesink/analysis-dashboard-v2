# DMD-22-01 — Upload a valid channel map from Assign Channels

Type: AFK

Blocked by: None

User stories covered: 1, 2, 5

## What to build

Add the narrow happy path for uploading an existing channel-map file from the scoped Assign Channels panel. A write user viewing a selected program/version can click **Upload**, choose a valid `channel_map.yml` or `channel_map.yaml`, and have the map applied to that current scope without uploading new CSV/RSP files.

The completed slice should be demoable end to end: open Assign Channels for a program/version with retained artifacts, upload a valid channel-map file, see the fixed plot mapping apply to the current scope, and see pending artifacts processed through the same retained-artifact processing behavior used by manual channel-map save.

## Acceptance criteria

- [x] Assign Channels renders an **Upload** button in the existing footer row with **Reset** and **Save**.
- [x] Clicking **Upload** opens a scoped upload popup for the current program/version.
- [x] The popup accepts a single valid channel-map file and submits it with the current `program_id` and `version`.
- [x] The backend exposes a scoped channel-map upload contract for an existing program/version.
- [x] The scoped upload contract parses and normalizes the YAML channel map using the existing channel-map normalization behavior.
- [x] A successful upload persists the channel map for the current program/version and processes retained pending artifacts through the existing retained-artifact processing path.
- [x] The UI shows a success result with processed/failed counts consistent with manual channel-map save feedback.
- [x] The Assign Channels panel refetches after success so the table displays the uploaded map values.
- [x] Focused tests cover the valid upload flow from API contract through UI callback behavior.
