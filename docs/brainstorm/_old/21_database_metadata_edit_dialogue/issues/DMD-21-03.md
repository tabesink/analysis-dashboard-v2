# DMD-21-03 — Harden cross-section dirty state, permissions, refresh, and accessibility

Type: AFK

Blocked by: DMD-21-02

User stories covered: 4, 5, 6

## What to build

Make the two-section metadata dialog production-ready for Assign Channels by extending dirty-close handling, permissions, post-save refresh, and accessibility to cover both Edit Metadata and Assign Channels. The dialog should be safe for real channel-map edits from the Database table and should not silently discard changes or leave stale channel-map state visible after save.

## Acceptance criteria

- [ ] Saving channel maps from the dialog uses the existing channel-map save contract.
- [ ] Successful channel-map saves invalidate all channel-map-dependent client queries used by the Database table, Dashboard filters, and Edit Metadata workflow.
- [ ] The Database table reflects saved channel-map changes without requiring a manual page refresh.
- [ ] Channel-map-required indicators update after a successful save when applicable.
- [ ] Users without write permission cannot save channel maps from the dialog and receive the same disabled affordance or explanation used on the full-page route.
- [ ] Closing the dialog with dirty metadata or channel-map values prompts using the established discard-confirm pattern; dirty changes are not silently lost.
- [ ] Pending scope changes while the dialog is open prompt when either section is dirty.
- [ ] Section switching does not prompt while the dialog stays open, even when a section has unsaved edits.
- [ ] Focus is trapped in the dialog while open, Escape/close behavior is predictable, keyboard users can reach both nav items and save actions, and initial focus remains sensible.
- [ ] Tests cover cross-section dirty-close, permission-gated channel-map save, successful save invalidation, and accessible nav switching.
- [ ] Manual verification covers opening multiple version rows in sequence and confirming each dialog open uses the newly clicked scope in both sections.
