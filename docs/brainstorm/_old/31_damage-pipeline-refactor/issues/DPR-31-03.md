## This slice changes

Finalize worker/task persistence behavior to align with simplified vocabulary:

- Task states: `validating`, `calculating`, `completed`, `failed`
- Cell states: `current`, `error`, `unavailable`

Persist explicit per-cell error/unavailable outcomes when channel data is missing or fails, and ensure task failure is visible through task status/reporting without hidden repair semantics.
