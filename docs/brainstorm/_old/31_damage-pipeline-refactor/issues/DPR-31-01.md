## This slice changes

Implement the schedule upload/save command contract so a valid schedule can immediately start (or reuse) the calculation lifecycle and return explicit status to UI callers. Invalid schedules must return validation/prerequisite reports and must not start calculation.

The slice is vertical: backend command behavior, API response shape, and frontend schedule flow wiring that renders command outcome.

## This slice must not rework

- Inspect Damage read-model behavior.
- Legacy backfill endpoint removal.
- Damage worker persistence internals beyond what is needed to return command status.

## Next slice can assume

- A single command entry point exists for schedule save/upload lifecycle start decisions.
- UI has explicit command outcome and task identity/state from that command.

## Acceptance criteria

- [x] Valid schedule save/upload returns a calculation lifecycle outcome (`calculating` path) and a task reference.
- [x] Invalid schedule save/upload returns a structured validation/prerequisite report and starts no calculation task.
- [x] API and frontend contract for schedule command outcome is covered by tests.
- [x] Existing auth/write permissions remain enforced on schedule mutation.
