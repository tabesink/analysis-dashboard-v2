- `../PRD.md`

## Previous slice provides

- Inspect API is read-only and returns selected rows + task status metadata.

## This slice changes

Update Inspect Damage frontend behavior to consume the read-only inspect model:

- remove automatic backfill/calculation triggers on page load
- render selected rows immediately even with unavailable cells
- show explicit running/failed status banners
- keep persisted `current`/`error`/`unavailable` cell presentation clear

Keep existing selection/filter/table/plot behavior otherwise stable.

## This slice must not rework

- Backend calculation orchestration policy.
- Fatigue math and persistence internals.
- Large UI redesign unrelated to status clarity.

## Next slice can assume

- Inspect page no longer acts as hidden mutation entry point.
- UI states match simplified vocabulary and are regression-tested.

## Acceptance criteria

- [x] Inspect page performs no backfill mutation call on load.
- [x] Selected event rows render even when damage cells are missing/unavailable.
- [x] Running calculation state appears as explicit banner/copy.
- [x] Failed calculation state appears as explicit banner/report entry point.
- [x] Frontend tests cover no-backfill behavior and running/failed status rendering.
