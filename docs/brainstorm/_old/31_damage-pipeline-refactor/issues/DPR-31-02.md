## Previous slice provides

- Schedule save/upload command can validate and return lifecycle outcome.

## This slice changes

When a new schedule is accepted for a program/version scope, clear prior persisted damage results for that scope before recalculation. Ensure repeated schedule saves for the same scope reuse/prevent duplicate active tasks.

Include end-to-end checks that the command path clears stale rows and preserves one-active-task invariant.

## This slice must not rework

- Inspect read response semantics.
- Frontend Inspect Damage UI behavior.
- Fatigue damage formula details.

## Next slice can assume

- Accepted schedule always starts from a clean scope result set.
- Task dedupe rules are enforced at command boundary.

## Acceptance criteria

- [x] Accepted schedule clears or invalidates prior damage rows for the same scope before task execution.
- [x] Repeated schedule save while task is active does not create duplicate active tasks.
- [x] Boundary tests prove cleanup + dedupe behavior via public command path.
- [x] No unauthorized caller can trigger cleanup/recalc.
