# AC-25-01 — Shell-mount channel reprocess modal and fix stacking

Type: AFK

Blocked by: None

User stories covered: 9, 10, 22, 25, 29

## What to build

Move channel reprocess modal rendering from the metadata editor to the Database page shell, matching how the import upload operation modal is hosted. Ensure the derived-data operation modal stacks above Edit Metadata so progress is not hidden behind the larger editor dialog.

Remove duplicate modal mount from the metadata editor while keeping scoped store wiring. Apply a shared shell operation-modal z-index convention so derived-data modals are not trapped at the same layer as Edit Metadata.

## Acceptance criteria

- [ ] Channel reprocess progress modal is rendered once on the Database page shell, not inside `MetadataEditDialog`.
- [ ] Only one modal instance exists for a running channel reprocess task.
- [ ] Modal is visibly above Edit Metadata when both are open.
- [ ] Modal open state still driven by channel reprocess store scoped to program/version.
- [ ] Close, dismiss-to-background, and summary close behaviors unchanged.
- [ ] Damage calculation modal hosting follows the same shell/stacking convention if it shares the modal component.
- [ ] Client tests cover shell mount and absence of duplicate modal markup when metadata editor is open.
