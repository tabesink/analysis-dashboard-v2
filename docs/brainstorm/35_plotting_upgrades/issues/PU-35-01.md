## Previous slice provides

- Documentation confirms there is no version selector in PRD-35.
- Reference and Target each compare exactly one program/version scope.

## This slice changes

Update Inspect Damage Reference/Target selection behavior so each side can only contain events from one program/version scope at a time.

Recommended behavior: first selected event establishes that side's program/version scope. Selecting another event in the same scope adds/removes it normally. Selecting a different scope clears that side and starts the new scope, with clear UI copy or toast feedback.

## This slice must not rework

- Global Filters.
- Dashboard route Load Data behavior.
- Damage calculation APIs.
- Plot rendering.
- Channel selection placement.

## Next slice can assume

- `comparison.reference.selected_event_ids` contains events from at most one program/version.
- `comparison.target.selected_event_ids` contains events from at most one program/version.
- Scope labels can be derived from selected event IDs and the event catalog without a version selector.

## Acceptance criteria

- [x] Reference selection cannot silently mix program/version scopes.
- [x] Target selection cannot silently mix program/version scopes.
- [x] Multiple events within the same selected scope remain selectable.
- [x] Clearing one side resets its active scope.
- [x] Existing selected event pruning still works when events disappear.
- [x] Tests cover same-scope add/remove, different-scope replacement, and empty-state reset.
- [x] Dashboard Load Data selection behavior is unchanged.
