# PPU-27-02 — Route schedule saves through precompute decisions

Type: AFK

Blocked by: PPU-27-01

User stories covered: 1, 3, 8, 9, 10, 11, 17, 25, 27, 28, 29, 30, 32, 33, 34, 37, 39, 40

## What to build

Extend durability schedule upload/replacement and schedule-row save so they use the post-upload precompute orchestrator after persisting schedule data. When channel and damage prerequisites are current, schedule save should start or reuse the existing `damage_calculation` task. When channel prerequisites are missing or stale, the schedule should still save successfully and the automatic damage step should be blocked without starting a doomed task.

This keeps workflow order independent: users may save schedule data before channel assignment finishes, and the later channel completion path from `PPU-27-01` can complete the damage step.

## Acceptance criteria

- [x] Schedule upload/replacement persists editable schedule rows before any precompute decision is made.
- [x] Schedule-row save persists repeats, weight, event matching, and multiplier before any precompute decision is made.
- [x] Schedule save starts a `damage_calculation` task when prerequisites are current.
- [x] Schedule save reuses an active derived-data task for the same program/version instead of starting a duplicate.
- [x] Schedule save succeeds without starting damage calculation when channel prerequisites are missing or stale.
- [x] Blocked automatic precompute produces a concise blocked decision that can be converted to toast-only feedback.
- [x] Invalid schedule rows continue to fail damage validation rather than producing partial damage results.
- [x] Server tests cover schedule upload, schedule-row save, active-task reuse, blocked missing-channel prerequisites, invalid schedule rows, and successful damage start.
