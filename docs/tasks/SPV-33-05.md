# SPV-33-05 — Aggregate comparison plot data and delta metrics

## Behavior added or changed

- Added a pure aggregation module that converts raw comparison facts into all v1 plot families:
  - cumulative damage by program/version
  - absolute-by-event channel rows
  - cumulative damage by channel
  - target-vs-reference channel deltas
- Added value-mode-aware selected outputs so each family exposes both absolute and normalized values while returning the currently selected value from `comparison.value_mode`.
- Added delta metrics on channel comparisons: signed absolute delta, percent difference, ratio, and normalized ratio.
- Added low-reference guard semantics so ratio-style metrics return `null` (not misleading finite values) when reference damage is missing, zero, or below threshold.

## Interfaces changed

- Added `client/src/features/inspect-damage/lib/build-damage-comparison-aggregates.ts`:
  - `buildDamageComparisonAggregates()`
  - `DamageComparisonAggregateInput`
  - typed aggregate row interfaces for each plot family
  - explicit metadata contract for normalized denominators and low-reference threshold

## Tests added and what they prove

- Added `client/src/features/inspect-damage/__tests__/build-damage-comparison-aggregates.test.ts` covering:
  - cumulative program/version grouping by dataset/program/version with absolute and normalized selected outputs
  - event-channel and channel cumulative groupings suitable for grouped/stacked event bars and channel summaries
  - signed delta math (`target - reference`), percent-difference and ratio formulas, normalized-ratio formula, and low-reference guard behavior for missing/zero reference channels

## Follow-on assumptions for next slice

- SPV-33-06 can consume one deterministic pure output for all comparison plot families and render mode-aware legends/subtitles directly from returned metadata.
- Plot wiring can rely on `selected_value`/`selected_metric` from aggregates for the active mode without recomputing formulas in UI components.

## Decisions intentionally left unchanged

- No chart rendering or route wiring changes.
- No changes to comparison state persistence or raw-fact construction contracts.
- No backend aggregation or fatigue-calculation behavior changes.
