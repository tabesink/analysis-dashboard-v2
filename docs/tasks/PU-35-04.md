# PU-35-04 — Add shared damage scale transform and in-plot toggle

Date: 2026-06-15

## Behavior added or changed

- Added a shared `applyDamageScale(value, mode)` utility for Inspect Damage plotting.
- The focused 3D path now applies damage-scale transforms through this shared utility instead of inline `Math.log10(1 + x)` logic.
- Moved the `Normal` / `Log` damage-scale toggle into the main plot area as a subtle top-right control.
- Removed damage-scale toggle ownership from the left overlay rail.

## Interfaces changed

- Added `client/src/features/inspect-damage-3d/lib/damage-scale.ts` exporting:
  - `applyDamageScale(value, mode)`
- `DamagePlotView` now uses `applyDamageScale` for cell damage transforms before layout calculation.
- `DamagePlotOverlayControls` no longer accepts `onDamageScaleModeChange` and no longer renders damage-scale controls.

## Tests added and what they prove

- `client/src/features/inspect-damage-3d/__tests__/damage-scale.test.ts` verifies:
  - linear mode returns raw finite non-negative values.
  - log mode applies `log10(1 + value)`.
  - negative and non-finite values are clamped to `0`.
- `client/src/features/inspect-damage-3d/components/DamagePlotView.test.tsx` verifies:
  - the `DamagePlotView` surface renders a `Damage scale` radiogroup with `Normal` and `Log` options in the main plot area.

## Follow-on assumptions for the next issue

- 2D plot specs can reuse `applyDamageScale` for consistent linear/log behavior.
- `DamagePlotView` already owns local `damageScaleMode` state that can be passed into upcoming 2D spec builders.

## Decisions intentionally left unchanged

- Plot layout math (`computeDamagePlotLayout`) and damage aggregation contracts are unchanged.
- Existing sidepanel `Plot Inputs` ownership for channels and value mode remains unchanged.
- Version-slice behavior in this slice remains as-is; removal is left for later PU-35 slices.
