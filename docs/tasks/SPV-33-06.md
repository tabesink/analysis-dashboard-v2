# SPV-33-06 — Wire comparison UI, plots, regressions, and docs

## Behavior added

- Added a dedicated `Comparison` central tab on Inspect Damage.
- Wired side-panel comparison controls for:
  - Reference Load Data selection
  - Target Load Data selection
  - Plotted channel selection
  - Absolute/normalized value mode
- Comparison inspect requests now use the union of selected Reference and Target event IDs.
- Comparison content now renders v1 plot families from aggregate outputs:
  - cumulative damage by program/version
  - damage by event/channel
  - cumulative damage by channel
  - Target-vs-Reference channel deltas
- Added comparison subtitle/legend context for mode, selection counts, low-reference guards, and excluded invalid/missing data.

## Interfaces changed

- `useInspectDamageState()` now returns:
  - `comparison`
  - `updateComparison(patch)`
- New helper module:
  - `client/src/features/inspect-damage/lib/build-damage-comparison-view-model.ts`
  - exports `getComparisonInspectEventIds()` and `buildDamageComparisonViewModel()`
- `InspectDamageCentralTabSwitcher` now supports a `comparison` tab key.

## Tests added

- `client/src/features/inspect-damage/__tests__/build-damage-comparison-view-model.test.ts`
  - validates union inspect IDs, missing-selection empty states, and v1 aggregate view-model output context.
- `client/src/features/inspect-damage/components/InspectDamageCentralTabSwitcher.test.tsx`
  - validates comparison tab rendering in the central tab switcher.

## Regression checks run

- `npm run test -- build-damage-comparison-view-model.test.ts InspectDamageCentralTabSwitcher.test.tsx`
- `npm run test -- inspect-damage-view-state.test.ts build-damage-comparison-raw-facts.test.ts build-damage-comparison-aggregates.test.ts`

## Follow-on assumptions

- v1 comparison rendering is currently card/list style output from aggregate series; chart-specific visual polish can iterate without changing comparison state/fact/aggregate contracts.
- Existing single-pool table and inspect tabs remain the canonical non-comparison flow and continue to use `data_state.selected_event_ids`.

## Decisions intentionally unchanged

- No fatigue recomputation in the browser.
- No backend pre-aggregation endpoint added in this slice.
- `aggregation_event_scope` behavior remains `selected_only` default with no new UI for `all_in_program_version`.
