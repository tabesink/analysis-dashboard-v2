# IDM-28-05 — Recovery, partial recalc policy, docs

Completed 2026-06-11.

## Summary

Closed the recovery loop for program/versions with mixed damage populations (some events `current`, others `error`/missing) after per-event channel resolution landed in IDM-28-03/04.

## Changes

- **`server/services/scope_damage_repair.py`** — `assess_scope_damage_repair_state()` returns `complete`, `needs_recalc`, or `stale_only` for a scheduled scope.
- **`server/services/post_upload_precompute.py`** — Inspect backfill and channel-reprocess follow-up use repair state instead of blocking on any persisted row.
- **`server/services/damage_inspect.py`** — Scope state exposes `needs_damage_repair` for client backfill planning.
- **Client** — `resolveInspectDamageBackfillScopes` includes scopes flagged `needs_damage_repair` (mixed cohort repair).
- **Docs** — Canonical column-index + per-event header lookup rule and operator recovery steps in `docs/architecture/derived-data-upload-pipeline.md` and `docs/brainstorm/28_inspect_dmg_fix/HANDOFF.md`.

## Partial repair policy

| State | Auto backfill |
|-------|---------------|
| All scheduled channels `current` | No-op |
| Mixed `current` + `error` / missing | Start full recalculation |
| Stale-only | No-op (still inspectable) |

Manual **Calculate Damage** always starts when prerequisites pass (unchanged).
