# P14-13 — Bootstrap admin after runtime DB switch

Date: 2026-06-09  
Status: DONE

## Scope

Prevent admin lockout after `Create New Database` or `Connect Database` runtime switch actions.

## Implementation notes

1. Updated runtime store initialization path in `server/routers/export.py`:
   - after migrations initialize the target DB store, call `UserService(store, settings).bootstrap_admin()`
   - then run health checks and proceed with swap.
2. Added regression coverage in `tests/server/routers/test_export_router.py`:
   - create a new database,
   - logout,
   - verify admin can log in again using configured `admin_secret` credentials.

- New regression test ensures post-switch admin login remains functional.
