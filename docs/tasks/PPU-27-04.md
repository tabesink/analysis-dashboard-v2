# PPU-27-04 — Backfill missing damage from Inspect Damage for write users

**Status:** DONE (2026-06-11)

## Summary

Inspect Damage now treats completely missing persisted damage as repairable state for write users. The inspect read path stays persisted-only; automatic backfill starts or reuses `damage_calculation` through the post-upload precompute orchestrator and surfaces progress through the existing derived-data modal.

## Implementation

- `server/services/post_upload_precompute.py`
  - `decide_after_inspect_damage_access()` returns `no_op`, `blocked`, `start_damage_calculation`, or `reuse_active_task` for missing damage only; stale rows remain manual repair.
  - `inspect_precompute_decision_to_response()` maps decisions to `DamageCalculateResponse` fields.
- `server/routers/damage.py`
  - `POST /api/v1/damage/backfill` (write users only) routes through the inspect-access orchestrator.
- `client/src/features/inspect-damage/lib/`
  - `resolve-inspect-damage-backfill-scopes.ts` selects missing scopes with ready prerequisites.
  - `plan-inspect-damage-backfill-attempts.ts` prevents duplicate session attempts and active-task collisions.
  - `apply-inspect-damage-backfill.ts` reuses `active_damage_task_id` or calls the backfill endpoint.
- `client/src/app/inspect-damage/page.tsx`
  - Auto-backfill effect after inspect load for write users; toast-only blocked feedback; derived-data modal via existing damage-calculation store.

## Tests

- `tests/server/services/test_post_upload_precompute.py`
  - Ready prerequisites start damage; blocked prerequisites; active-task reuse; no-op when current or stale rows exist.
- `tests/server/routers/test_damage_router.py`
  - Backfill starts task when damage missing; blocked prerequisite report; write access required.
- `client/src/features/inspect-damage/__tests__/`
  - Backfill scope resolution, attempt planning, and apply behavior.
- `client/src/lib/api/damage.test.ts`
  - Backfill API client posts to `/damage/backfill`.

## Notes

- Manual Calculate Damage still uses `POST /damage/calculate` and can start work for stale scopes.
- Inspect `/damage/inspect` remains read-only with no synchronous fatigue compute.
