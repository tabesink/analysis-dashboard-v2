# P5-38 Dashboard load-data selection is channel-map gated

## Summary

Dashboard load-data side panels now prevent selecting any program/version/event scope that does not have a channel map. Existing selections are pruned automatically when an event becomes non-selectable.

## Implementation Notes

- In `server/services/query.py`, `selectable_for_plotting` for real events is now derived from channel-map presence (`has_channel_map`), instead of always true.
- In `client/src/components/dashboard/shared/HierarchicalEventTree.tsx`, program and version checkboxes are disabled when they have no selectable descendants.
- Program indeterminate behavior now uses true partial selection state from selectable descendants.
- In `client/src/components/dashboard/side-panel/LoadDataSection.tsx`, a prune pass removes any selected event IDs that are no longer selectable in the latest event catalog.
