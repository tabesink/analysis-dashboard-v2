# PU-35-09 — Add program/version comparison card

## Behavior added

- Extended `buildDamage2DPlotSpec` to support `cumulative_by_program_version` with `program-version-bar` chart metadata.
- Program/version card subtitle now compares selected scope labels directly: `Reference {program/version} vs Target {program/version}`.
- Program/version values now respect selected value mode (`absolute`/`normalized`) and damage scale (`linear`/`log`) through the shared spec builder path.
- Added scope-aware empty-state handling for missing Reference/Target scope selections and for missing aggregate rows.
- Routed `cumulative_by_program_version` through the real grouped-bar card renderer in `Damage2DPlotCard` instead of placeholder shell output.

## Interfaces changed

- `buildDamage2DPlotSpec` input now accepts optional scope metadata:
  - `referenceScopeKey?: string | null`
  - `targetScopeKey?: string | null`
  - `referenceScopeLabel?: string | null`
  - `targetScopeLabel?: string | null`
- Existing callers remain compatible; when explicit scope metadata is omitted, the builder derives labels from `aggregates.program_version`.

## Tests added

- `client/src/features/inspect-damage-3d/__tests__/build-damage-2d-plot-spec.test.ts`
  - validates `cumulative_by_program_version` values and subtitle scope labels,
  - validates empty-state messaging when a side scope is missing.
- `client/src/features/inspect-damage-3d/components/DamagePlotView.test.tsx`
  - validates the program/version card renders in the 2D grid with selected scope subtitle labels and bar output.

## Follow-on assumptions for next slice

- `PU-35-10` can rely on `cumulative_by_program_version` being a real rendered card in the 2D grid.
- Plot subtitles can assume one-scope-per-side selection semantics from `PU-35-01`.

## Decisions intentionally unchanged

- No backend/API changes.
- No sidepanel control ownership changes.
- No version dropdown reintroduction.
