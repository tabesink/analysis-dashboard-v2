## P9-06: Production Docker config

Date: 2026-05-15
Status: DONE

### Objective

- Provide a repeatable, junior-friendly production deployment path for a trusted local network.
- Keep deployment settings env-primary while preserving existing YAML defaults/templates.
- Surface app and database schema version/status without cluttering normal workflows.

### Changes

- Added the original `Dashboard/deployment/` build-on-host deployment assets.
- Added root `.env.example` for local development and LAN defaults.
- Updated `server/config.py` so deploy-facing environment variables explicitly override YAML init kwargs.
- Updated `client/Dockerfile` to stop baking `http://localhost:8000` as the default API URL.
- Updated `/api/v1/info` with runtime mode, database connectivity, and DB schema version/target status.
- Updated `/health/ready` to return HTTP 503 when storage is unavailable.
- Updated the header version label to include DB schema status.
- Converted `server/settings.yaml` to a development template and moved real deployment secrets to env files.
- Updated `scripts/start-server-lan.sh` to derive LAN CORS origins from the current hostname.
