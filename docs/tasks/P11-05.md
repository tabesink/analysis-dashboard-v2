# P11-05: Database nested event tree + Edit Events mixed-null save fix

Date: 2026-04-16
Status: DONE

## Goal

Replace the flat per-event table on the Database page with a nested
`Program ID > Version > Event` tree, remove the stale `display*` / `meta:*`
column-key indirection that caused metadata columns to render as dashes, and
fix the Edit Events save path so that a program/version group whose events are
partially `null` can actually propagate the chosen metadata value.

Related decisions: DEC-030 (nested tree + raw-column lookup), DEC-031 (split
draft/baseline state for mixed-null metadata groups).

## Scope

1. Database page tree UI
   - Three-level shadcn `Collapsible` tree: Program ID > Version > Event.
   - `chevron-down` toggle; programs default-expanded, versions default-collapsed.
   - Indent guide lines with darkest-to-lightest gray shading ramp per level.
   - Parent rows show child count and rolled-up status pill.
   - `Version` and `Event` column headers removed; values nest under `Program ID`.

2. Semantic tightening at the right level
   - Status pill rendered only at the version level; removed from event rows.
     `Status` remains a filterable column header at program level.
   - Delete is scoped to program and version nodes only. Event-level checkboxes
     and the select-all control are removed; individual events cannot be deleted
     from this view.

3. Remove `display*` / `meta:*` indirection
   - `DatasetRow` aliased to `DatasetInfo`.
   - `toDatasetRow`, `displayKeyToServerColumn`, and all `display*` /
     `meta:*` column keys deleted.
   - Column keys become raw DB column names
     (`program_id`, `version`, `status`, `suspension_component`,
     `axle_location`, `drive_type`, `material_construction`,
     `steering_position`, `damper_type`, `vehicle_type`, `gvw`, `fgawr`,
     `rgawr`, ...).
   - `getColumnValue` is a direct lookup:
     `(dataset as unknown as Record<string, unknown>)[columnKey]`.

4. Edit Events save propagation (mixed-null groups)
   - `buildProgramVersionDraftValues(options, events)` now returns
     `{ draft, baseline }` instead of a single map.
   - For each metadata field:
     - All events share one non-empty value -> `draft = baseline = value`.
     - Some events have the value and others are `null` (mixed-null) ->
       `draft = value`, `baseline = ''`. The user sees the existing value; a
       Save click produces a non-empty diff and propagates the value to every
       event in the group.
     - Multiple distinct values or all empty -> `draft = baseline = ''`.
   - Caller destructures the new shape and sets `draftValues` /
     `baselineDraftValues` separately.

## Changes Implemented

### Database page tree

- `client/src/components/upload/DatabaseEventTree.tsx` (new)
  - Groups `DatasetInfo[]` by `program_id` then `version`.
  - Renders nested `Collapsible` nodes with chevron toggles.
  - Rolled-up version status from child events; program status rolled up from
    its versions.
  - Indent guide lines and level-based gray shading.
- `client/src/app/database/page.tsx`
  - Integrates `DatabaseEventTree`.
  - Removes `toDatasetRow`, `displayKeyToServerColumn`, and `display*` /
    `meta:*` column keys.
  - `columnFilters`, `visibleColumns`, and `staticColumnDefinitions` use raw
    DB column names.
  - `getColumnValue` simplified to a direct property read on `DatasetInfo`.
  - `dynamicMetadataColumns` drops the `meta:` prefix and uses `config.column`
    as the column key.
  - Header row is a separate flex container aligned to the tree body.
- `client/src/types/upload.ts` (no change) continues to own `DatasetInfo` as
  the single source of truth for column keys.

### Edit Events save path

- `client/src/app/database/edit/page.tsx`
  - `buildProgramVersionDraftValues` refactored to return
    `{ draft: MetadataDraftValues; baseline: MetadataDraftValues }`.
  - Introduces a `resolveField(key, rawValues)` helper that implements the
    three cases above.
  - The effect that loads a program/version group now destructures the new
    shape and calls `setDraftValues(nextDraftValues)` /
    `setBaselineDraftValues(nextBaselineValues)` separately.
  - No changes to the `PUT /api/v1/dashboard/program-version/metadata`
    contract or to `QueryService.update_program_version_metadata`.

## Validation

- IDE lint diagnostics for changed files: no errors.
- Manual verification:
  - Database page: programs default-expanded, versions default-collapsed,
    chevron toggles work, indent lines and shading ramp render correctly,
    metadata columns populate with their raw DB values (dashes only when the
    underlying cell is actually `null`).
  - Status pill appears at version level only; no status pill on event rows;
    `Status` filter in the header still works.
  - Delete actions available on program and version rows only.
  - Edit Events: selecting a program/version whose events are partially `null`
    for a field now shows the existing value in the form, and clicking `Save`
    issues the `PUT` call, propagates the value to every event in the group,
    and the Database tree shows the value after reload.
