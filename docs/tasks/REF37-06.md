## REF37-06 - Contributor edit and delete policy naming

Date: 2026-06-16

### Behavior Changed

- Extracted explicit uploaded-data policy names for route-level ownership semantics:
  - contributor edit policy for channel-map and schedule commands
  - uploaded-data admin policy
  - scope delete policy
- Channel-map and schedule command routes now return a shared contributor edit forbidden detail when write users target datasets they do not own.
- Scope delete route now uses a named scope-delete forbidden detail while preserving existing owner-or-admin behavior.
- Added explicit channel-map role behavior coverage:
  - non-owner write users are forbidden from uploading channel maps for another contributor's program/version.
  - admins can upload channel maps for another contributor's program/version.

### Interfaces Changed

- No endpoint shapes changed.
- Added named policy helpers and permission detail constants in `server/upload/policies.py`:
  - `has_uploaded_data_admin_policy`
  - `has_contributor_edit_uploaded_data_policy`
  - `has_scope_delete_uploaded_data_policy`
  - `EDIT_UPLOADED_DATA_FORBIDDEN_DETAIL`
  - `SCOPE_DELETE_FORBIDDEN_DETAIL`
- Dashboard and upload routers now call these helpers instead of invoking store permission methods inline.

### Tests Added

  - `test_upload_program_version_channel_map_forbids_non_owner_write_user`

### Follow-on Assumptions For Next Slice

- Contributor ownership and admin override semantics are now named consistently across channel-map, schedule, and scope delete route checks.
- Route-level role behavior coverage now explicitly includes channel-map non-owner denial and admin override, so database administration slices can rely on stable uploaded-data edit/delete contracts.
