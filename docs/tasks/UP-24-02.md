# UP-24-02 — Show channel reprocess progress in Assign Channels

Date: 2026-06-11
Status: DONE

## Summary

Assign Channels save and channel-map YAML upload now open a reusable derived-data progress modal that polls the `channel_reprocess` task from UP-24-01. Closing the modal leaves processing running; the scoped Edit Metadata dialog shows an inline banner with Reopen progress.

## Implementation

- Added client types `DerivedTaskStartResponse` and `DerivedTaskStatusEvent`.
- Added `derivedDataApi.waitForDerivedDataTask` mirroring upload task polling with retry semantics.
- Pure modules:
  - `derived-task-progress.ts` — coarse phase bands + server live message mapping
  - `channel-reprocess-completion.ts` — success/warning/failure summary builder
- `channel-reprocess-store.ts` — scoped task tracking, background polling, modal/banner state, active-task reuse without duplicate polls.
- UI:
  - `DerivedDataOperationModal` — close-only progress (no cancel); summary on completion
  - `DerivedDataProgressPanel` — validating / extracting / generating stepper
  - `ChannelReprocessBanner` — scoped inline banner in `MetadataEditDialog`
- `AssignChannelsPanel` starts tasks via updated save/upload helpers (no immediate query invalidation).
- Query invalidation runs on task completion via existing `invalidateQueriesAfterChannelMapSave`.
