# P1-19 — Unified release version sync and changelog workflow

Date: 2026-03-30  
Status: DONE

## Goal

Ensure one product version is used across client and server metadata, and that the frontend always displays the real running client/server versions without version drift.

## Implementation Notes

1. Added `scripts/release_version.py`:
   - Validates SemVer input.
   - Updates root `VERSION`.
   - Updates `client/package.json` `version`.
   - Updates `server/pyproject.toml` `[project].version`.

2. Added `scripts/check_version_sync.py`:
   - Compares root `VERSION` with client/server mirrored versions.
   - Returns non-zero on mismatch for CI and local checks.

3. Updated `client/next.config.ts`:
   - Removed `package.json.version` usage.
   - Reads version from root `VERSION` during build config evaluation.

4. Added CI guard:
   - `.github/workflows/version-sync.yml` runs `python3 scripts/check_version_sync.py` on PRs and pushes to `main`/`master`.

5. Added release process documentation:
   - `docs/release-versioning.md` defines canonical files, commands, and release checklist.
   - `CHANGELOG.md` updated under `Unreleased` with versioning workflow changes.

## Verification

- `python3 scripts/release_version.py 1.0.0` executed successfully.
- `npm --prefix client run generate:version` executed to refresh generated client version artifact.
- `python3 scripts/check_version_sync.py` passes with `Version sync check passed: 1.0.0`.
