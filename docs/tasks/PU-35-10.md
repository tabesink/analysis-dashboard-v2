# PU-35-10 - Signed target-delta card

Date: 2026-06-15

## Behavior added

- Added `target_delta_vs_reference` support to the 2D plot spec builder using signed absolute deltas (`target - reference`) from `channel_delta`.
- Added selected-channel filtering for target-delta specs while preserving sidepanel channel order.
- Added low-reference warning propagation into card-level warning copy and per-channel flags.
- Added a new diverging SVG card renderer with a visible zero baseline, signed bars, semantic delta legend, and warning badge support.
- Added tooltip payload fields for channel label, reference damage, target damage, signed delta, and current value mode.

## Interfaces changed

- `Damage2DPlotSpec` now includes optional `deltaRows` metadata used by delta-card tooltip rendering.
- `buildDamage2DPlotSpec()` now returns concrete specs for `plotType: 'target_delta_vs_reference'` instead of the generic placeholder empty state.
- `Damage2DPlotCard` now routes `target_delta_vs_reference` to `TargetDeltaVsReferencePlotCard`.

## Tests added/updated

- Updated `build-damage-2d-plot-spec` unit tests to cover:
  - signed positive/negative delta rendering data
  - zero-only channel handling
  - low-reference warning/flag propagation
  - selected-channel filtering for `channel_delta`
- Added `TargetDeltaVsReferencePlotCard.test.tsx` component tests covering:
  - diverging bars + zero baseline marker
  - legend labels
  - tooltip content fields
  - low-reference warning display
- Updated `DamagePlotView.test.tsx` fixture and assertion to verify the target-delta card renders real diverging content (not placeholder copy).

## Follow-on assumptions for next slice

- `PU-35-11` can reuse the signed-delta card scaffold and add card-local metric selection (percent difference / ratio semantics) without replacing card layout structure.
- `deltaRows` metadata is now available as the canonical per-channel tooltip source for delta cards.

## Decisions intentionally unchanged

- Ratio and percent-difference UI/metric toggling remains out of scope for this slice.
- Focused 3D renderer behavior is unchanged in this issue; this work is scoped to the 2D target-delta card path.
