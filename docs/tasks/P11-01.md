## P11-01: Frontend plot pipeline deep module

Date: 2026-03-30  
Status: DONE

### Goal
Unify duplicated plot fetch/decode/transform behavior used by lazy interactive fetch and grid sequential streaming into one reusable boundary module.

### Implementation
- Added `client/src/lib/plot-pipeline.ts` with:
  - `fetchAndDecodePlot(...)`
  - `toSVGPlotCurvesData(...)`
  - `toErrorMessage(...)`
- Refactored `use-lazy-plot-fetch.ts` to call `fetchAndDecodePlot(...)`.
- Refactored `use-sequential-plot-data.ts` to call the same shared function.

### Boundary Impact
- Plot hooks now own orchestration state (abort, loading, retries), while binary transport/decoding details are hidden behind one module.
- Changes to binary endpoint or decode path now occur in one place instead of two hooks.

### Verification
- IDE lint diagnostics report no issues in changed frontend files.
- Runtime behavior remains hook-compatible: same inputs/outputs, same loading/error semantics.

### Files Changed
- `client/src/lib/plot-pipeline.ts`
- `client/src/hooks/use-lazy-plot-fetch.ts`
- `client/src/hooks/use-sequential-plot-data.ts`
