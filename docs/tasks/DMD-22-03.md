# DMD-22-03 — Upload parity, refresh, and cache behavior

**Status:** DONE (2026-06-10)

## Scope

Prove Assign Channels channel-map upload matches Upload Data YAML normalization, refreshes the same dependent client queries as manual save, clears the missing-map warning after success, preserves Reset/Save behavior, and surfaces differentiated success/failure feedback.

## Implementation

### Shared process feedback

- Added `channel-map-process-feedback.ts` with `formatChannelMapProcessMessage`, `getChannelMapProcessToastTone`, and `resolveChannelMapUploadAttempt`.
- Messages distinguish:
  - processed retained files;
  - no pending files to process;
  - partial or total processing failures (warning toast, failed count preserved).
- Upload popup stays open only when the upload request throws; successful API responses close the dialog even when some artifacts fail.

### Assign Channels integration

- `AssignChannelsPanel` uses the shared feedback helper for both Save and Upload.
- Upload still refetches `channel-map-editor` after invalidating shared database/dashboard query keys.
