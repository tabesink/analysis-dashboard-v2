## CU40-01 - Upload task lifecycle state and heartbeat

### Behavior added

- Extended shared active upload-task statuses to include `cancelling`.
- Added durable upload-task lifecycle metadata columns in schema:
  - `started_at`
  - `cancel_requested_at`
  - `finished_at`
  - `last_heartbeat_at`
  - `runner_id`
- Added request-time stale heartbeat reconciliation in `UnifiedStore.reconcile_stale_upload_tasks(...)` that marks stale active upload-task rows `failed` with fail-closed messaging.
- Wired stale reconciliation into:
  - database switch/delete operation admission
  - folder upload task polling and SSE routes
  - active derived-task reuse lookup for program/version scope.
- Extended operation blocker payloads with `last_heartbeat_at` and `cancel_requested_at`.
- Extended upload task status payloads with lifecycle timing fields and `runner_id`.
- Worker task update calls now refresh heartbeat metadata during long-running folder upload, channel reprocess, and damage calculation workflows.

### Interfaces changed

- `server/upload/task_kinds.py`
  - `ACTIVE_TASK_STATUSES` now includes `cancelling`.
- `server/storage/database.py`
  - `UnifiedStore.create_upload_task(...)` accepts optional `runner_id`.
  - `UnifiedStore.update_upload_task(...)` accepts lifecycle and heartbeat metadata fields.
  - Added `UnifiedStore.reconcile_stale_upload_tasks(...)`.
- `server/services/operation_admission.py`
  - blocker payloads now include heartbeat/cancel timing metadata and admission reconciles stale upload rows before blocking.
- `server/models/upload.py` + `server/routers/upload.py`
  - upload task response now includes lifecycle timestamps and `runner_id`.

### Tests added/updated

- Updated task-kind contract test for `cancelling` active status.
- Updated schema initialization tests to assert lifecycle column presence.
- Updated startup reconciliation test to include `cancelling` rows and `finished_at` stamping.
- Added route-level test proving stale active upload rows are reconciled before database switch admission blocks.
- Added upload polling tests for lifecycle timestamp/runner visibility and stale heartbeat reconciliation during status polling.

### Assumptions for next slice

- Cancel endpoints in `CU40-02` should set `status='cancelling'`, `cancel_requested_at`, and refresh `last_heartbeat_at` through existing task-update helpers.
- Cooperative cancellation checkpoints can rely on lifecycle metadata already present in polling/admission payload contracts.
