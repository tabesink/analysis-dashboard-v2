# PU-35-03 - Shared plot card shell

Date: 2026-06-15

## Behavior added or changed

- Added `PlotCardShell` as a reusable chart-card chrome component for arbitrary renderer children.
- `PlotCardShell` supports:
  - Bottom-left title chip with optional subtitle and optional prefix indicator.
  - Loading state.
  - Error state with message.
  - Empty state with title/description copy.
  - Top-right action slot and footer slot.
- Refactored `SVGPlotCard` to consume `PlotCardShell` for loading, error, empty-axes, and normal render paths.

## Interfaces changed

- New component: `client/src/components/charts/PlotCardShell.tsx`.
- New export: `PlotCardShell` from `client/src/components/charts/index.ts`.
- `SVGPlotCard` public props remain unchanged.

## Tests added and what they prove

- Added `client/src/components/charts/PlotCardShell.test.tsx` with component-level coverage for:
  - Normal rendering of children plus title/subtitle/action/footer slots.
  - Loading state rendering and suppression of normal renderer children.
  - Error state rendering and suppression of normal renderer children.
  - Empty state rendering and suppression of normal renderer children.
- Re-ran `ComparisonLoadDataSections` regression tests to verify nearby PU-35 sidepanel behavior remains intact.

## Follow-on assumptions for the next issue

- `PU-35-04` can place shared Normal/Log scale controls on top of consistent card chrome semantics.
- Future 2D cards can reuse `PlotCardShell` without introducing damage-specific coupling.

## Decisions intentionally left unchanged

- `SVGPlotCard` still renders empty plot axes (via `SVGPlot` with empty curves) when no curves exist.
- Sync toggle and action affordances remain in `SVGPlotCard`; they were not moved into global chart state.
