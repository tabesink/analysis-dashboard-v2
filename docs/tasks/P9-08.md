## P9-08: Mode-driven network exposure + production config guards

Date: 2026-03-10
Status: DONE

### Objective
- Expose frontend/backend to the network only in production mode.
- Keep development mode localhost-only by default.
- Support production mode for non-container runs via `server/settings.yaml`.
- Enforce production-grade security checks at startup.

### Changes
- Updated `server/config.py`:
  - Added `app_env` setting (`development` or `production`).
  - Added mode defaults:
    - `development`: `host=127.0.0.1`, `debug=true`, `auth_cookie_secure=false`
    - `production`: `host=0.0.0.0`, `debug=false`, `auth_cookie_secure=true`
  - Added production validation gates:
    - reject `debug=true`
    - reject localhost bind host
    - require secure auth cookie
    - reject wildcard CORS origin
    - require `jwt_expiry_hours <= 24`
    - require non-placeholder `jwt_secret` with minimum length
- Updated `server/settings.yaml`:
  - Added `app_env: development` for non-container mode switching.
  - Set default host to `127.0.0.1` for local safety.
- Updated `server/settings.docker.yaml`:
  - Added `app_env: production`.
- Updated `client/package.json` scripts:
  - `dev` binds to `127.0.0.1`.
  - `start` binds to `0.0.0.0` (production exposure).
- Updated Docker/Compose deployment config:
  - Added `APP_ENV=production` for server.
  - Made frontend API URL configurable via `NEXT_PUBLIC_API_URL` / hostname-derived LAN fallback.

### Verification
- Development:
  - run frontend with `npm run dev` and confirm localhost-only bind.
  - run backend with `app_env: development` and confirm localhost bind.
- Production (non-container):
  - set `app_env: production`, valid `jwt_secret`, strict CORS origin(s).
  - run backend and verify network reachability + startup security enforcement.
  - run frontend with `npm run build && npm run start` and verify remote access.
- Production (container):
  - set deployment env values before `docker compose up -d`.
  - confirm frontend backend calls resolve from remote clients.
