# P8-03: Add `data_version` monotonic counter

## Summary

Added a monotonic `data_version` counter to support multi-user cache
coordination groundwork.

## Changes

- Updated `UnifiedStore.write_connection()` in `server/storage/database.py` to
  bump `data_version` on successful commit by default.
- Added internal `_increment_data_version_unlocked(conn)` helper that stores
  `data_version` in `_schema_metadata`:
  - inserts `1` on first write
  - increments by `+1` on subsequent writes
- Added `UnifiedStore.get_data_version()` for consumers that need to read the
  current counter value.
- Added `bump_data_version=False` override for non-mutating maintenance flows:
  - `export_to_parquet()` checkpoint transaction
  - `vacuum()`
- Added storage behavior coverage:
  - `test_unified_store_data_version_increments_on_write`
