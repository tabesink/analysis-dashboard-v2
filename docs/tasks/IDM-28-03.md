# IDM-28-03 — Damage lookup uses per-event headers

**Status:** DONE (2026-06-11)

## Summary

Wired `EventHeaderProvider` and `resolve_plot_channels_from_headers` into `QueryService.get_damage_channel_series` so schedule-driven damage reads each event's plot axis channel names from stored header metadata before querying `measurements_raw`. Legacy generic `col_N` pattern matching remains as fallback when headers are unavailable.

## Changes

- `server/services/query.py` — per-event header resolution in damage channel series lookup; units filled from resolved header metadata when the channel map has no unit.
- `tests/server/services/test_damage_query_service.py` — mixed Moog + abbreviated fixture batch, header-resolution vs empty-measurement error distinction.
- `tests/server/services/test_damage_calculation_task.py` — integration test proving damage task persists 12 `current` rows per event in a mixed batch.

## Behavior

1. For each damage spec, load the plot row from `dim_channel_map` and event headers via `EventHeaderProvider`.
2. When headers exist, resolve lookup channel name (and unit when missing) from column indices; persist resolver errors (`missing_headers`, `column_out_of_range`) on unavailable series items.
3. When headers are missing and the map uses generic `col_N`, fall back to legacy pattern matching against distinct raw channel names.
4. When headers are missing and the map uses explicit titles, use the persisted title string (backward compatibility).
