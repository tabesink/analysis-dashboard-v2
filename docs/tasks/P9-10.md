# P9-10 Upload Scope-Delete Ownership and Cache Router Tests

## Status

DONE (2026-05-15)

## Summary

Added route-level data-safety coverage for `POST /api/v1/upload/program-version/delete`. The tests prove the public API enforces the same owner/admin boundary already covered at the store level and refreshes event-related cache groups after successful hard deletes.

## Changes

- Added `tests/server/routers/test_upload_router.py`.
- Covered write-enabled owner deletion for a fully owned program/version scope.
- Covered mixed ownership returning HTTP 403 for a non-admin writer without deleting either event.
- Covered admin deletion of the same mixed-owner scope.
- Covered cache behavior: successful deletes invalidate `events`, `event_count`, `program_ids`, and `versions`, while forbidden deletes leave those entries untouched.
