# P14-11 — Create/connect runtime database actions

Date: 2026-06-09  
Status: DONE

## Scope

Add immediate runtime database management actions from the Database page:

- Admin-only **Create New Database** with typed confirmation.
- Write user/admin **Connect Database** with typed confirmation and selection.
- Health-gated runtime switch with rollback path visibility.

## Implementation notes

1. Added new export-router endpoints:
   - `GET /api/v1/export/database/list`
   - `POST /api/v1/export/database/create-new`
   - `POST /api/v1/export/database/connect`
2. Implemented managed DB catalog + filename validation (`dashboard*.db` only, no path components).
3. Implemented typed confirmation checks on backend:
   - `CREATE NEW DATABASE`
   - `CONNECT <database_name>`
4. Implemented create flow:
   - generate `dashboard-<timestamp>.db`
   - initialize/migrate via `MigrationRunner.initialize_store_for_startup()`
   - run health checks
   - swap `app.state.db` + `app.state.session_manager` only after checks pass
5. Implemented connect flow to existing managed DB with the same health-gated swap logic.
6. Added frontend API methods in `client/src/lib/api/export.ts` for list/create/connect.
7. Updated Database side panel UI:
   - new admin row above export for **Create New Database**
   - **Connect Database** action for write users/admins
8. Updated database page handlers to:
   - collect typed confirmations
   - list/select DB to connect
   - execute create/connect and reload page on success
9. Added router tests for:
   - unauthenticated rejection on create/connect
   - admin create-and-switch success
   - write-user list-and-connect success
