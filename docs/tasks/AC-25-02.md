# AC-25-02 — Modal-only feedback for Assign Channels save and upload

**Status:** DONE (2026-06-11)

## Summary

Removed transitional `toast.loading` / `toast.dismiss` from Assign Channels save and channel-map YAML upload success paths. The derived-data progress modal is now the sole in-flight feedback channel after the start API returns a `channel_reprocess` task id.

## Changes

- Added `assign-channels-reprocess-flow.ts` to orchestrate save/upload start → immediate `trackChannelReprocessTask` without toast noise.
- Updated `AssignChannelsPanel` to call the flow helper; kept `toast.error` for validation/API failures and `toast.success` for reset/pre-reset restore.
- Active-task reuse still passes `reused_existing_task` into the store so duplicate polls are not started.

## Tests

- `assign-channels-reprocess-flow.test.ts` — save/upload open tracking immediately; active-task reuse; no track on start failure.
- Existing coverage retained: `channel-reprocess-store.test.ts`, `derived-data-cross-flow.test.ts`, `DerivedDataOperationModal.test.tsx`, `channel-reprocess-completion.test.ts`.

## Not in this slice

- Database-page background banner when Edit Metadata is closed (AC-25-03).
