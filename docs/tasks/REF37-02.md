# REF37-02 — Server Upload Policy Contract

## Behavior added or changed

- Extracted folder-upload lane rules into a pure policy module at `server/upload/policies.py`.
- Folder-upload filename classification is now centralized and explicit for:
  - CSV-only and RSP-only data batches
  - CSV/RSP mixed-batch rejection
  - optional `channel_map.yaml` / `channel_map.yml` companion detection
  - unsupported file ignore behavior
- `POST /api/v1/upload/folder/start` now consumes the pure policy helper instead of embedding lane rules directly in the router parse loop.

## Interfaces changed

- New pure module: `server/upload/policies.py`
  - `classify_upload_filenames(filenames: list[str]) -> UploadBatchClassification`
  - `require_data_files(classification: UploadBatchClassification) -> None`
  - Stable lane constants:
    - `FOLDER_UPLOAD_TASK_KIND`
    - `FOLDER_UPLOAD_PHASE_UPLOAD_RECEIVED`
    - `FOLDER_UPLOAD_PHASES`
    - `NO_DATA_FILES_ERROR`
    - `MIXED_DATA_FORMAT_ERROR`
- Router usage:
  - `server/routers/upload.py::_parse_upload_payload` now calls policy helpers for file-lane classification and validation.
  - `server/routers/upload.py::_build_upload_task_event` now uses the shared upload default phase constant.

## Tests added

- `tests/server/unit/routers/test_upload_policies.py`
  - CSV-only batch classification
  - RSP-only batch classification
  - mixed CSV/RSP rejection
  - optional `channel_map.yaml` / `channel_map.yml` companion detection
  - unsupported-file ignore behavior + no-data rejection contract
  - stable task-kind and phase-name constants
- Existing route guard regression kept green:
  - `tests/server/routers/test_upload_router.py::test_folder_upload_start_forbids_read_only_user_before_parsing_or_task_creation`

## Follow-on assumptions for next slice

- Client-side parity work (`REF37-03`) can consume the same policy vocabulary:
  - CSV/RSP exclusivity
  - optional channel-map companion semantics
  - unsupported-file handling contract
- Server route URL and successful start response shape are unchanged.

## Decisions intentionally unchanged

- Ingestion service internals, DB operations, and byte staging behavior remain unchanged.
- Router file split and broader orchestration extraction remain out of scope for this slice.
