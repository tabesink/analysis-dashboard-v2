# P11-07: Database Schema DDL Ownership Consolidation

## Summary

Moved DuckDB schema ownership into `server/schema.yaml` while keeping `UnifiedStore` as the connection owner and runtime facade. The first slice was DDL-only: no repository extraction, no schema doctor, and no data-backfill extraction.

## Changes

- Normalized `server/schema.yaml` into a named-map registry for sequences, tables, columns, constraints, additive columns, and indexes.
- Updated `SchemaLoader` to generate idempotent DuckDB DDL from the normalized registry.
- Added `SchemaApplier` as the shared DDL application path.
- Changed `UnifiedStore._init_schema()` to delegate declared DDL to `SchemaApplier`, then run the existing data backfills.
- Updated `MigrationRunner.apply_initial_schema()` to use the same applier as startup.
- Added storage behavior tests for fresh initialization, additive column upgrades, and migration/startup table consistency.
- Updated `docs/database-schema.txt` to document `server/schema.yaml` as the declared schema source.
