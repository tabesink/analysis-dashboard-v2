# OP39-01 — Remove Database Import Remnants

## Status

DONE (2026-06-17)

## Behavior Changed

- Removed the remaining runtime database-import lane while preserving the supported admin lane:
  - `database export`
  - `database create`
  - `database connect/switch`
  - `database delete`
- Legacy import endpoints remain as compatibility tombstones and return `410 Gone` with replacement guidance.
- Export task polling now uses a single export stall threshold path.
- Health readiness checks no longer include import-in-progress bypass behavior.

## Interfaces Changed

### Server

- Removed import-only service internals from `server/services/export.py`:
  - import task creation/execution, upload staging registry, and import validation API internals.
- Removed import runtime/config surface:
  - `server/config.py` `duckdb_import_*` settings and env overrides.
  - `server/upload/task_kinds.py` removed `database_import` remnant constant.
- Kept and preserved:
  - export task start/poll/cancel/download contracts.
  - create/connect/delete database admin contracts.
  - transfer-package internals outside the removed UI lane.

### Client

- Removed import-only API helpers and inference helper from `client/src/lib/api/export.ts`.
- Removed import state from UI stores and sync/auth behavior:
  - `client/src/stores/ui-store.ts`
  - `client/src/hooks/use-data-version-sync.ts`
  - `client/src/stores/auth-store.ts`
- Reduced database operation UI/hook to export-only flow:
  - `client/src/hooks/use-database-operation.ts`
  - `client/src/components/upload/DatabaseOperationModal.tsx`

## Tests Added/Updated

- Updated server route tests to assert import remains unavailable (`410`) while export/create/connect/delete contracts remain intact.
- Replaced import-specific stall/progress route expectations with export-only expectations.
- Reduced export-service tests to active export lifecycle behavior and restart reconciliation.
- Updated upload policy unit tests to remove deleted `database_import` task-kind constant expectations.
- Updated frontend API tests to export-only helper surface and messages.

## Focused Verification Run

- `uv run pytest tests/server/routers/test_export_router.py tests/server/services/test_export_service.py tests/server/unit/routers/test_upload_policies.py`
- `pnpm vitest src/lib/api/export.test.ts` (run from `client/`)

## Residual Risks

- Historical task JSON files that were persisted with `kind="import"` are no longer expected in active code paths; if present from older snapshots, they will not be re-created by current runtime.
- Legacy docs under archival brainstorm folders still mention import behavior intentionally as historical context.
