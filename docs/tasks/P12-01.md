# P12-01: Closed-registration auth model + admin user management + `can_write` permission tier

Date: 2026-04-22
Status: DONE

This task note covers the entire Phase 12 implementation (P12-01 through
P12-08); the per-row table in `docs/master-build-plan.md` is the canonical
breakdown.

Related decision: DEC-032.
Plan: `.cursor/plans/admin-settings-and-permissions_6df9da97.plan.md`.

## Goal

Replace the open auto-create login flow with a closed model where:

- Login requires an existing `users` row + bcrypt password.
- `admin_secret` only seeds the initial admin row at startup, then is never
  consulted at login again.
- A new `can_write` permission tier sits between `admin` and read-only `user`.
- Admins manage the user roster from a new `/settings/users` page.
- Users can self-register, but new accounts default to read-only.
- Read-only users see Database / Edit Filters as disabled in the sidebar and
  are redirected away from those routes.
- Per-version `Status` writes remain admin-only.

## Backend

### Schema migration (`server/storage/database.py`)

`users` table gains two columns:

```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS can_write BOOLEAN DEFAULT FALSE
ALTER TABLE users ADD COLUMN IF NOT EXISTS last_settings_visit_at TIMESTAMP
UPDATE users SET can_write = TRUE WHERE role = 'admin' AND can_write IS NOT TRUE
```

`docs/database-schema.txt` is updated to document both new columns.

### `UserService` (`server/services/user.py`, new)

Owns the entire user lifecycle so `AuthService` can shrink to a JWT/credential
verifier:

- `bootstrap_admin()` — startup-only. If no admin row exists, hashes
  `settings.admin_secret` (or stores as-is if already a bcrypt hash) and
  inserts the admin with `can_write=TRUE`.
- `list_users()` / `get_user(id)` / `delete_user(id)`.
- `create_user(username, password, role, can_write)` — used by both the
  admin-create flow and `/auth/register`.
- `update_user(id, role?, can_write?)` — promoting to `admin` forces
  `can_write=TRUE`.
- `reset_password(id, new_password)` (admin-driven) and
  `change_own_password(id, current_password, new_password)`.
- `get_pending_count(admin_id)` and `mark_settings_visited(admin_id)` for
  the sidebar notification dot.
- `verify_credentials(username, password)` used by `AuthService.authenticate`.

### `AuthService` (`server/services/auth.py`)

Refactored to a thin wrapper:

- `authenticate` looks up the user via `UserService.verify_credentials`,
  returns the row or raises `AuthenticationError`.
- `_authenticate_admin` and the auto-create branch are removed entirely.
- JWT payload now includes `can_write`.

### Models

- `server/models/auth.py`
  - `LoginRequest.password` is required (`min_length=8`).
  - New `RegisterRequest` (username + password) and `ChangePasswordRequest`
    (current_password + new_password).
  - `CurrentUserResponse` adds `can_write: bool`.
- `server/models/user.py` (new) — `UserListItem`, `CreateUserRequest`,
  `UpdateUserRequest`, `ResetPasswordRequest`, `PendingCountResponse`.

### Routers

- `server/routers/auth.py`
  - `POST /auth/login` is now strictly closed (no auto-create).
  - Adds `POST /auth/register` (public, strict rate limit).
  - Adds `POST /auth/change-password` (authenticated).
  - `_set_auth_cookie` helper used by login + register.
  - `GET /auth/me` re-reads from DB so role/permission changes are reflected
    immediately.
- `server/routers/admin_users.py` (new) — admin-only CRUD:
  - `GET /admin/users`, `POST /admin/users`, `PATCH /admin/users/{id}`,
    `DELETE /admin/users/{id}`.
  - `POST /admin/users/{id}/reset-password`.
  - `GET /admin/users/pending-count`, `POST /admin/users/mark-visited`.
  - Self-update guard (admin cannot demote self) and self-delete guard.
- `server/main.py` — includes the new router; `lifespan` startup hook calls
  `UserService(...).bootstrap_admin()`.

### Permission dependency (`server/dependencies.py`)

```python
async def require_write_or_admin(user: AuthenticatedUserDep) -> AuthenticatedUser:
    if user["role"] == "admin" or user.get("can_write"):
        return user
    raise HTTPException(status_code=403, detail="Write access required")
```

`AuthenticatedUser` TypedDict gains `can_write: bool`. `WriteUserDep` and
`UserServiceDep` aliases added. `require_write_or_admin` replaces
`require_admin` on:

- `server/routers/upload.py` — `purge_deleted_events`.
- `server/routers/dashboard.py` — `reset_filter_options`,
  `create_or_update_custom_field`, `update_program_custom_field_values`,
  and `update_program_version_metadata` (the `status` field inside the
  payload is still gated on `is_admin` inside `QueryService`).

### Rate limiting (`server/middleware/rate_limiter.py`, `server/config.py`)

Adds `register` (strict) and `auth` (login/me/change-password) rate-limit
categories with configurable per-minute caps in `RateLimitingSettings`.

## Frontend

### API client + auth store

- `client/src/lib/api/auth.ts` — `LoginRequest.password` required;
  `RegisterRequest` and `ChangePasswordRequest` interfaces; `CurrentUser`
  gains `can_write`; `authApi.register` and `authApi.changePassword`.
- `client/src/lib/api/client.ts` — generic `patch<T>()` helper.
- `client/src/lib/api/users.ts` (new) — `usersApi.list / create / update /
  remove / resetPassword / pendingCount / markVisited`.
- `client/src/lib/api/index.ts` — re-exports.
- `client/src/types/user.ts` (new) — `UserRole`, `AdminUser`,
  `CreateUserPayload`, `UpdateUserPayload`, `ResetPasswordPayload`,
  `PendingCount`.
- `client/src/stores/auth-store.ts` — `register` action, `changePassword`,
  `refresh`, plus `selectIsAdmin` and `selectCanWrite` selectors
  (`canWrite` = `role === 'admin' || can_write === true`).

### Sidebar + nav gating

- `client/src/types/layout.ts` — `NavigationItem` gains `requirePermission?:
  'write' | 'admin'` and `disabledTooltip?`.
- `client/src/config/sidebar-config.ts` — `Database` and `Edit Filters`
  tagged with `requirePermission: 'write'` and a "Read-only access" tooltip.
- `client/src/components/layout/NavMain.tsx` — when permission is unmet,
  renders a disabled `<button>` (50% opacity) with the read-only tooltip
  instead of the `<Link>`.
- `client/src/components/layout/AppSidebar.tsx` — Settings icon as the LAST
  item inside `<SidebarContent>` (above the footer separator). For admins it
  links to `/settings/users` and shows a small notification dot when
  `usersApi.pendingCount()` > 0; for non-admins it renders disabled with an
  "Admin only" tooltip. Clicking the icon fires `usersApi.markVisited()` and
  re-fetches the count.

### Login page (`client/src/app/login/page.tsx`)

Wrapped in shadcn `<Tabs>`:

- **Sign in** — username + password (both required, password min length 8).
- **Register** — username + password + confirm password. Validates
  `password.length >= 8` and `password === confirm` client-side, calls
  `authStore.register(...)`, then redirects to `/dashboard`.

Both flows redirect to `/dashboard` on success (read-only users no longer
have access to `/database`).

### Settings page (`client/src/app/settings/users/page.tsx`, new)

Monochrome shadcn shell (`bg-card`, `border-border`, `text-muted-foreground`,
`bg-muted/50` for table header).

- Route guard redirects non-admins to `/dashboard`.
- Top-of-page `Card` for "Change my password" using
  `authStore.changePassword`.
- "Create user" button opens a `Dialog` with username, password, role
  select, and write toggle.
- shadcn `<Table>` columns:
  `Username | Password | Role | Write access | Created | Actions`.
  - Password column shows `••••••••` plus a "Set" / "Not set" status hint
    and a "Reset" button that opens a dialog with a single new-password
    field.
  - Role uses `<Select>` (User / Admin).
  - Write access uses `<Switch>`; for admin rows it is forced ON and
    disabled.
  - Actions column uses `<DropdownMenu>`; "Delete user" opens an
    `<AlertDialog>` for confirmation.
- Admin's own row hides Delete and disables Role/Write changes.
- Self-registered rows whose `created_at > last_settings_visit_at` get a
  muted "New" badge until the next page load.
- Calls `usersApi.markVisited()` on mount.

### Route guards

- `client/src/app/database/page.tsx` and
  `client/src/app/database/edit/page.tsx` — existing unauthenticated
  redirect now sits next to a `if (!canWrite) router.replace('/dashboard')`
  guard. The `Status` field gate inside `/database/edit` is unchanged
  (still keyed on `isAdmin`).

### shadcn primitives

`dialog`, `label`, `badge`, `switch` were missing from
`client/src/components/ui/`. They were added as small local components
matching the existing shadcn style instead of pulling new Radix
dependencies (`tabs`, `alert-dialog`, `dropdown-menu`, `popover`, `select`,
`tooltip`, `card`, `table`, `input`, `button` already exist).

## Tests

- `tests/server/services/test_user_service.py` — bootstrap_admin idempotence,
  closed-login rejection of unknown user, register flow creates read-only
  user, change_own_password requires correct current password.
- `tests/server/routers/test_auth_routes.py` — `/auth/login` rejects
  passwordless, `/auth/register` creates a `can_write=False` user and sets
  the cookie, `/auth/change-password` rejects wrong current password.
- `tests/server/routers/test_admin_users_router.py` — `require_admin`
  enforcement on `/admin/users`, admin can reset another user's password,
  `require_write_or_admin` allows admin and write-tier users only on
  upload/dashboard mutate endpoints.
