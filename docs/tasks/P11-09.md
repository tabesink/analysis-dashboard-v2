# P11-09: Data backfill extraction from schema initialization path

## Summary

Extracted startup row-mutating backfills from `UnifiedStore._init_schema()` into a
dedicated storage module while preserving execution order and behavior.

## Changes

- Added `server/storage/data_backfills.py` with `apply_startup_backfills(conn)`.
- Moved existing startup backfill SQL from `server/storage/database.py` into that
  helper without changing SQL semantics:
  - admin `users.can_write` normalization
  - legacy `dim_event.maturity -> status` copy when the legacy column exists
  - null-to-false normalization for `dim_event` boolean phase columns
- Kept `_init_schema()` flow unchanged: declared DDL apply first, then backfills.
- Added `test_unified_store_runs_extracted_startup_backfills_idempotently` in
  `tests/server/storage/test_schema_initialization.py`.
- Updated `docs/database-schema.txt` to point runtime backfill ownership at
  `server/storage/data_backfills.py`.
