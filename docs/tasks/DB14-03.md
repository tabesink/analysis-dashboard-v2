# DB14-03: Channel-map snapshot normalization across YAML and UI

**Status:** DONE (2026-06-09)

## Summary

Added a normalization and storage path so YAML-uploaded and UI-authored channel maps converge on one immutable snapshot model with authoring provenance, one active snapshot per program/version, and per-event snapshot lineage.

## Changes

- `server/services/channel_map_snapshot.py` — `ChannelMapNormalizationService` and `ChannelMapSnapshotStorageService` (`artifact://snapshots/...`).
- `server/schema.yaml` — `channel_map_snapshots`, `active_channel_map_snapshots`, `event_ingestion_links.channel_map_snapshot_id`.
- `server/storage/database.py` — snapshot upsert/list/active helpers, scope-delete cleanup, portability exclusions.
- `server/services/ingestion.py` — persist and activate snapshots on YAML ingest and UI channel-map save; link events to snapshots.
