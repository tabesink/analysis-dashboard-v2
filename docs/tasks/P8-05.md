# P8-05: Frontend data version polling + query invalidation

## Summary

Implemented frontend polling for `/api/v1/sync/version` and invalidation of
sync-sensitive queries when `data_version` advances.

## Changes

- Added `client/src/lib/api/sync.ts` with `syncApi.getVersion()`.
- Added `client/src/hooks/use-data-version-sync.ts`:
  - polls `['sync-version']` every 10 seconds when authenticated
  - tracks last seen version in a ref
  - invalidates sync-sensitive queries when version increases:
    - `['all-events']`
    - `['program-ids']`
    - `['versions']`
    - `['filter-options']`
    - `['event-catalog']`
  - resets local version baseline when user is unauthenticated
- Wired polling in `client/src/app/providers.tsx` with `useDataVersionSync()`.
- Exported API surface in `client/src/lib/api/index.ts` and hook barrel in
  `client/src/hooks/index.ts`.
- Added API regression test:
  `client/src/lib/api/sync.test.ts`.
