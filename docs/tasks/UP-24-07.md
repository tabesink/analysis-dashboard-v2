# UP-24-07 — Preserve Canonical Raw During Channel Reprocess

Status: DONE (2026-06-11)

## Summary

Assign Channels no longer recomputes `measurements_raw`. Upload/canonicalization writes all numeric signal columns, excluding index/time, into `measurements_raw` as canonical full-resolution analytical data. Channel reprocess now reads those raw rows and regenerates only `measurements_lttb` plus channel-map lineage.

## Implementation Notes

- `DataTransformer.transform_to_long` ignores plot mappings and stores all numeric signal columns after the timestamp column.
- Uploads without a channel map now still create event lineage with `measurements_status=current` and `lttb_status=absent`.
- Channel reprocess validates canonical raw exists, deletes/reinserts LTTB rows only, and emits validation/generation progress phases.
- Channel-map snapshot stale handling no longer marks `measurements_status` stale.
- Existing legacy databases with incomplete raw should be repaired through a one-time canonical-artifact backfill path, not normal Assign Channels reprocess.
