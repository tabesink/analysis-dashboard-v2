# P11-04 - Minimal DB Hardening

Date: 2026-03-30  
Status: DONE

## Scope

Apply a minimal hardening pass that protects database behavior while dashboard and route features continue to evolve:
- Keep routers transport-focused by removing direct DB reach-through.
- Consolidate upload dataset read SQL into a dedicated service boundary.
- Align frontend event loading with backend filter semantics.
- Tighten frontend session payload types to backend contracts.
- Add focused regression tests for DB behavior and user scoping.

## Changes Implemented

### 1) Dashboard router/service boundary
- `server/routers/dashboard.py`
  - Removed local `_resolve_usernames_by_id` helper and direct `query_service.db` usage.
  - Events response now consumes username fields prepared by service.
- `server/services/query.py`
  - `get_all_events()` now resolves and attaches `uploaded_by_username` and `last_updated_by_username`.

### 2) Upload dataset read boundary
- Added `server/services/upload_query.py`
  - New `UploadQueryService.list_datasets(limit, offset)` owns dataset count, facets, and paginated rows.
- `server/dependencies.py`
  - Added `get_upload_query_service` and `UploadQueryServiceDep`.
- `server/routers/upload.py`
  - `GET /upload/datasets` now uses `UploadQueryServiceDep` instead of inline SQL.

### 3) Frontend filter contract alignment
- `client/src/hooks/use-all-events.ts`
  - Hook now accepts `globalFilters` and includes them in request/query key.
- `client/src/hooks/use-event-catalog.ts`
  - Removed local event filtering logic over fetched rows.
  - Builds request filter payload from allowed fields and calls backend with those filters.

### 4) Session API contract tightening
- `client/src/types/session.ts`
  - Extended `UIPreferences` to match backend shape (`active_tab`, `color_grouping`).
  - Added `SessionCreatePayload` and `SessionUpdatePayload`.
  - Removed obsolete top-level `color_grouping` field from `SessionState`.
- `client/src/lib/api/session.ts`
  - `create`/`update` now use explicit payload contract types.

### 5) Regression tests
- Added `tests/server/services/test_boundary_regressions.py` with:
  - Event listing hides soft-deleted rows and includes username enrichment.
  - Upload dataset service returns paginated items and global facets.
  - Session manager enforces user-scoped access (`get`/`update` for non-owner fail).
