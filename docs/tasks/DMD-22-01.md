# DMD-22-01 — Valid channel-map upload happy path

**Status:** DONE (2026-06-10)

## Scope

Add the narrow happy path for uploading `channel_map.yml` or `channel_map.yaml` from Assign Channels for the currently selected program/version.

## Implementation

### Backend

- `POST /api/v1/dashboard/program-version/channel-map/upload` accepts `program_id`, `version`, and one `channel_map` file.
- Write/ownership guard matches `PUT /program-version/channel-map`.
- `IngestionService.upload_channel_map_yaml_and_process_artifacts` loads YAML via `ChannelMapLoader`, validates the fixed eight-plot contract, and reuses `_persist_channel_map_and_process_artifacts` (extracted from manual save).
- Basename helper: `server/utils/channel_map_file.py`.

### Frontend

- `AssignChannelsPanel` footer adds **Upload** between scope actions and **Reset**/**Save**.
- `ChannelMapUploadDialog` provides scoped file-drop UI; validates single-file basename client-side before submit.
- `uploadProgramVersionChannelMap` calls `dashboardApi.uploadChannelMap` and reuses `invalidateQueriesAfterChannelMapSave`.
- Basename helper: `features/edit-metadata/lib/channel-map-file.ts`.
