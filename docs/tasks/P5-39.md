# P5-39 Dashboard Pending Pseudo-Event Cleanup

## Summary

Fixed a dashboard regression where versions without channel maps were still checkable and where raw `__pending_channel_map__` pseudo-event IDs appeared as leaf labels in the Load Data tree.

## Implementation Notes

- Updated `/api/v1/dashboard/events` mapping in `server/routers/dashboard.py` to pass through:
  - `has_channel_map`
  - `missing_channel_map`
  - `selectable_for_plotting`
- Updated `client/src/components/dashboard/shared/HierarchicalEventTree.tsx` to hide pseudo-event leaves whose IDs start with `__pending_channel_map__::`.
- Left grouping and warning behavior intact at program/version level so pending-only scopes remain visible but disabled.
