# P5-32: Increase CSV upload timeout to 60 minutes

Date: 2026-03-30
Status: DONE

## Goal

Reduce interruption risk for large CSV folder uploads in a small local-network deployment by increasing client upload timeout, without changing authentication/session architecture.

## Implementation

- Added `CSV_UPLOAD_TIMEOUT_MS = 3_600_000` (60 minutes) in `client/src/lib/api/upload.ts`.
- Applied that timeout to `uploadApi.startFolderUpload(...)` via `postFormDataWithProgress(...)`.
- Kept auth expiry policy unchanged (24h production guard remains in place).

## Verification

- Lint check for `client/src/lib/api/upload.ts` passed.
- Confirmed timeout constant is used by the start-upload request path.
- Manual long-upload runtime verification requires interactive upload execution in the UI.
