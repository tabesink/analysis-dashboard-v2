# P5-35: Interactive viewer empty plot when no curves visible

Date: 2026-04-22
Status: DONE

## Goal

Replace the "No curves visible / Enable curves from the side panel" message in the Interactive viewer with an empty plot (axes only), matching the look of the empty card in the Grid view.

## Problem

When a plot was selected in the Interactive viewer but the user toggled off every curve in the side panel, the card showed a centered text block ("No curves visible / Enable curves from the side panel") instead of preserving the chart shape. This was visually inconsistent with the Grid view, where empty cards still render axes via `<SVGPlot curves={[]} ... />`.

## Implementation

In `client/src/components/dashboard/interactive-viewer/InteractiveViewer.tsx`:

- Removed the `visibleEventIds.length === 0` branch and its `NoCurvesState` sub-component.
- Removed the redundant `Loading plot...` fallback branch that was reached only when `curves` happened to map to empty.
- Collapsed the render-state ladder to: `loading -> error -> render InteractiveCanvasPlot with current curves`.
- Gated `PinnedEventsOverlay` on `curves.length > 0` so it does not appear over an empty plot.
- `PlotLabel` (bottom-left plot title) is unconditionally shown when a plot is selected.
- Removed now-orphaned `effectiveEventIds` and `visibleEventIds` memos (added in P5-30) since the empty-state gate they fed has been deleted.

`InteractiveCanvasPlot` already handles `curves=[]` safely: `calculateAxisLimits` returns its default empty range, the offscreen canvas and spatial grid are built empty, and the `SVGAxes` layer still renders.
