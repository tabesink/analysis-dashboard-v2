# DPR-31-06 — Reset migration + regression hardening + docs closeout

## Behavior added or changed

- Program/version scope hard-delete now also removes scoped derived-data lifecycle tasks (`damage_calculation`, `channel_reprocess`) so reset-first cleanup does not leave orphan running/failed task rows behind.
- Inspect Damage stale status badge copy now uses neutral operator wording (`Outdated`) instead of legacy stale terminology in table headers/cells.
- Migration reset notes now provide executable operator/developer steps aligned with the shipped scope-delete reset behavior.

## Interfaces changed

- `UnifiedStore.hard_delete_program_version_scope()` now deletes scoped rows from `upload_tasks` for derived-data damage lifecycle task kinds.
- No API contract shape changes were introduced; route contracts remain unchanged.

## Tests added and what they prove

- `tests/server/routers/test_upload_router.py`
  - `test_scope_delete_cleans_damage_schedule_and_scoped_damage_tasks` proves program/version scope delete resets persisted damage rows, active/archived durability schedules, and scoped derived-data damage tasks.
- Existing targeted regression tests were re-run for DPR-31 acceptance boundaries:
  - `tests/server/routers/test_durability_schedule_router.py::test_put_schedule_save_clears_scope_damage_and_reuses_active_task`
  - `tests/server/routers/test_damage_router.py::test_damage_inspect_is_query_only_and_skips_repair_prerequisite_checks`
  - `tests/server/routers/test_damage_router.py::test_damage_inspect_does_not_compute_when_persisted_rows_missing`
  - `client/src/features/inspect-damage/__tests__/plan-inspect-damage-backfill-attempts.test.ts`

## Follow-on assumptions for next issue

- Reset-first local rollout can rely on scope delete to clear schedule/damage/task artifacts for a program/version before reupload.
- Inspect Damage remains query-only on page load; lifecycle mutations stay command-owned.

## Decisions intentionally left unchanged

- Legacy `/api/v1/damage/backfill` endpoint remains for compatibility and explicit/manual mutation paths; inspect page-load behavior remains non-mutating.
- Fatigue calculation math, prerequisite evaluation, and task polling models remain unchanged.
