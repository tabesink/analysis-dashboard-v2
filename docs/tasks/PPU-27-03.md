# PPU-27-03 — Rescale scheduled damage for schedule-only edits

**Status:** DONE (2026-06-11)

## Summary

Added a synchronous rescale-only path for durability schedule edits that change only repeats, weight, or multiplier while event matching stays unchanged. When current persisted base damage exists for every scheduled event/channel row, scheduled damage is updated with `base_damage * repeats * weight * multiplier` without starting py-fatigue.

## Implementation

- `server/services/schedule_damage_rescale.py`
  - `is_schedule_scaling_only_change()` compares event matching and scaling inputs between previous and new preview JSON.
  - `assess_rescale_eligibility()` requires non-null base damage for every mapped channel on scheduled events; rejects error rows and rows stale for reasons other than `schedule_changed`.
  - `rescale_schedule_damage()` updates scheduled damage and row metadata (repeats, weight, multiplier, schedule identity, status, stale reason, timestamp) while preserving base damage.
- `server/services/post_upload_precompute.py`
  - `decide_after_schedule_save()` accepts optional `previous_preview` and returns `rescale_scheduled_damage` when eligible; otherwise falls back to existing damage-calculation start/reuse behavior.
- `server/routers/dashboard.py`
  - Schedule-row save captures pre-save preview and passes it into the precompute decision path.

## Tests

- `tests/server/services/test_schedule_damage_rescale.py`
  - Eligible scaling-only save rescales without creating a damage task and applies the formula.
  - Missing, stale (non-schedule), and error base damage fall back to full damage calculation.
  - Event-matching changes fall back to full damage calculation.
  - Blocked rescale preserves existing stale scheduled damage values.

## Notes

- Schedule attach/replace still always routes to full damage calculation (no previous scaling-only path).
- Invalid schedule rows during rescale fall back to damage calculation so whole-task validation behavior is unchanged.
