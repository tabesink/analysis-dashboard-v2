# DB14-04: Event preview metadata from canonical CSV

**Status:** DONE (2026-06-09)

## Summary

Added an `EventPreviewService` that derives lightweight preview metadata from canonical CSV bytes and stores one preview row per ingested event. CSV and RSP uploads converge on the same preview path because both use the canonical CSV produced during ingestion.

## Changes

- `server/services/event_preview.py` — derive headers, units, capped first-row sample, counts, warnings, and artifact references; persist via `event_previews`.
- `server/schema.yaml` — `event_previews` table (`event_id`, `preview_json`, `created_at`).
- `server/storage/database.py` — upsert/get helpers; scope delete and hard-delete cleanup; portability exclusion.
- `server/services/ingestion.py` — store preview at event commit for direct ingest and retained-artifact processing.
- `docs/database-schema.txt` — documents `event_previews`.
- `docs/decisions/log.md` — DEC-076 event preview contract.

## Tests

- `tests/server/services/test_event_preview.py` — derivation, persistence, CSV/RSP ingest integration, lightweight bounds, existing event retrieval.

## Preview JSON contract

```json
{
  "headers": ["..."],
  "units": ["..."],
  "first_rows": [["..."]],
  "row_count": 0,
  "column_count": 0,
  "warnings": [],
  "source_artifact_uri": "artifact://sources/...",
  "canonical_artifact_uri": "artifact://canonical/...",
  "source_filename": "event.csv",
  "conversion_kind": "identity"
}
```

`first_rows` is capped at 20 rows. Full canonical CSV bytes remain in filesystem derived artifacts only.
