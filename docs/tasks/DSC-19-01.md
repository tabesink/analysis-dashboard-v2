# DSC-19-01: Schedule attach/read API hardening

**Status:** DONE (2026-06-10)

## Summary

Closed test gaps on existing `POST/GET /api/v1/dashboard/program-version/schedule` endpoints and `DurabilityScheduleParser` edge cases so downstream DSC-19 slices can rely on stable API behavior. No route or schema changes.

## Changes

- `server/services/durability_schedule.py` — identical re-upload of the active schedule is a silent no-op (skips audit when active `schedule_id` unchanged).
- `tests/server/routers/test_durability_schedule_router.py` — GET 200/404; POST 400 for empty/non-`.sch`; admin cross-owner attach; identical-byte dedupe (`replaced_previous=false`, single attach audit).
- `tests/server/services/test_durability_schedule.py` — parser `*summary` boundary, missing `*id`/`*multiplier` defaults, zero-repeat entries; service-level identical checksum re-attach dedupe.

## Verified behavior

- GET returns active schedule context or 404 when none attached.
- POST rejects empty files and non-`.sch` extensions with 400.
- Admin may attach on program/versions they do not own; non-owner write users get 403.
- Identical checksum re-attach reuses the same `schedule_id` with `replaced_previous=false`, no replacement audit, and no duplicate attach audit when already active.
