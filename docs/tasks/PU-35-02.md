# PU-35-02 — Move plot inputs into Inspect Damage sidepanel

Date: 2026-06-15

## Behavior added or changed

- Added a new sidepanel `Plot Inputs` section directly below `Target Load Data` in Inspect Damage.
- Moved comparison channel selection and value mode controls from the 3D overlay rail into sidepanel controls owned by `DamageComparisonState`.
- Sidepanel channel toggles now enforce a one-channel-minimum invariant and show a toast if the user attempts to clear the final channel.
- `DamagePlotOverlayControls` now keeps only plot type, version slice, and damage scale controls.

## Interfaces changed

- Added `ComparisonPlotInputsSection` in `client/src/components/dashboard/side-panel/ComparisonPlotInputsSection.tsx`.
- `ComparisonLoadDataSections` now wires:
  - `onChannelToggle` => `onUpdateComparison({ selected_channel_keys })`
  - `onValueModeChange` => `onUpdateComparison({ value_mode })`
- `DamagePlotView` no longer handles value-mode or channel-toggle mutation paths.
- `DamagePlotOverlayControls` no longer accepts or renders value-mode/channel toggle controls.

## Tests added and what they prove

- Extended `ComparisonLoadDataSections` component tests to verify:
  - `Plot Inputs` renders after `Target Load Data`.
  - channel toggle updates `selected_channel_keys`.
  - final-channel deselect is blocked with user feedback.
  - value mode control updates `value_mode`.

## Follow-on assumptions for the next issue

- Sidepanel is now the canonical owner for persisted comparison channel selection and value mode.
- Plot overlay rail can be treated as non-owner for channel/value controls in upcoming 2D/3D layout slices.

## Decisions intentionally left unchanged

- Table `Cols` visibility remains table-local state and does not mutate plot channels.
- Plot rendering math and aggregation behavior remain unchanged in this slice.
- Overlay version slice and damage scale controls remain in place for later PRD-35 slices.
