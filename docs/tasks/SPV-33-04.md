# SPV-33-04 — Raw Reference/Target event x channel facts

## Behavior added or changed

- Added a pure comparison raw-fact builder that converts union inspect-damage response rows plus `DamageComparisonState` into deterministic event x channel facts.
- Facts are split by dataset membership (`reference` and `target`) based on selected event IDs; overlap events intentionally produce one fact per selected dataset.
- Builder filters to `comparison.selected_channel_keys` and only accepts canonical inspect-damage channels, so metadata keys do not become plot facts.
- Stale values remain in facts with explicit status metadata, while missing/null/invalid cells are excluded before aggregation.

## Interfaces changed

- Added `client/src/features/inspect-damage/lib/build-damage-comparison-raw-facts.ts`:
  - `buildDamageComparisonRawFacts()`
  - `DamageComparisonRawFactBuildResult`
  - `DamageComparisonRawFact`
  - `DamageComparisonExcludedFact`
  - exclusion reason/status types for downstream missing-data context

## Tests added and what they prove

- Added `client/src/features/inspect-damage/__tests__/build-damage-comparison-raw-facts.test.ts` covering:
  - overlap events appearing in both dataset memberships
  - selected-channel filtering that ignores non-canonical metadata keys
  - stale-value retention on facts plus explicit exclusion reasons for null/invalid cells

## Follow-on assumptions for next slice

- SPV-33-05 can consume `buildDamageComparisonRawFacts()` as the deterministic pre-aggregation input contract.
- Aggregation can rely on `facts` as usable rows and `excluded` for missing-data/quality context in legends and empty-state messaging.

## Decisions intentionally left unchanged

- No UI wiring or chart rendering changes.
- No backend API or fatigue-calculation behavior changes.
- No mutation of comparison session merge/persistence helpers.
