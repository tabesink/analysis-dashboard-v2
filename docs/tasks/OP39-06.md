## OP39-06 — Non-dismissible progress dialogs

### Behavior changed

- Active derived-data progress dialogs (`channel_reprocess`, `damage_calculation`) are now non-dismissible through modal close surfaces (`onOpenChange` close requests are ignored while `wizardStep === "progress"`).
- Derived-data progress footer copy/action that encouraged background dismissal was removed:
  - Removed “Closing this dialog does not cancel processing…”
  - Removed “Close and continue in background”
- Active upload and database export progress remain non-dismissible while active; terminal summaries remain closeable.
- Active cancel actions remain tied to real cancel paths:
  - Upload progress keeps `Cancel` (`onCancelUpload`)
  - Database export progress keeps `Cancel operation` (`onCancelOperation`)

### Interfaces changed

- `DerivedDataOperationModalProps` no longer accepts `onDismissProgress`.
- `DatabaseDerivedDataOperationModals` no longer wires progress-time dismiss callbacks to store modal-dismiss actions; close behavior now routes through modal-level active-state guard.

### Tests added/updated

- Updated `client/src/features/edit-metadata/__tests__/DerivedDataOperationModal.test.tsx`:
  - verifies progress UI no longer renders background-dismiss copy/action
  - verifies `onOpenChange(false)` is blocked during progress
  - verifies `onOpenChange(false)` is allowed in summary
- Added `client/src/features/database-upload/UploadOperationModal.test.tsx`:
  - verifies active upload progress blocks close requests
  - verifies active upload shows real cancel action
  - verifies summary allows close requests
- Added `client/src/components/upload/DatabaseOperationModal.test.tsx`:
  - verifies active export progress blocks close requests
  - verifies active export shows real cancel action
  - verifies summary allows close requests

### Residual risks / follow-ups

- The custom `AlertDialog` component currently models close through `onOpenChange(false)` from close button/backdrop. Escape-key behavior is not separately modeled in this component; if modal internals migrate to Radix dialog primitives, the same guard pattern should be preserved and explicitly regression-tested with keyboard events.
