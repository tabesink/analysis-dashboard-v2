# P2-14 Program/Version Hard-Delete Scope

## Summary

Database users can now delete a whole program or a single program/version even when the scope only contains pending channel-map artifacts and no processed events. The operation is a hard delete for the entire selected scope.

## Implementation Notes

- Added backend preview and delete helpers in `server/storage/database.py` that count events, measurements, custom field values, artifact rows, channel-map rows, owners, and registered artifact paths before deleting.
- Added `POST /api/v1/upload/program-version/delete` in `server/routers/upload.py` with writer/admin ownership checks. Writers must own every event/artifact in the scope; mixed ownership is blocked with 403.
- Updated the Database tree so program and version checkboxes select explicit scope keys, including pending-only versions with no event IDs.
- Updated Database delete handling to call the scope-delete endpoint for program/version selections and keep the existing event bulk-delete path for event-only selections.

## Verification

- Added regression coverage for pending-only version deletion, processed event deletion, program-level deletion across versions, and mixed ownership delete rules in `tests/server/services/test_ingestion_service_status.py`.
- Scope deletes invalidate dashboard/upload-derived caches through the upload router after successful deletion.
