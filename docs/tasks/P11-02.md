## P11-02: Frontend session/filter boundary helpers

Date: 2026-03-30  
Status: DONE

### Goal
Reduce hidden coupling across `useSession` and `useFilterState` by centralizing session storage/sync primitives and selection-diff logic.

### Implementation
- Added `client/src/lib/session/session-sync.ts` containing:
  - key constants and timing constants
  - session ID/backup read/write helpers
  - update merge + non-empty update checks
  - `hasUnrenderedSelection(...)` comparison logic
- Updated `use-session.ts` to consume these shared helpers.
- Updated `use-filter-state.ts` to use the shared unrendered-change detector.

### Boundary Impact
- Hook behavior is now composed from reusable primitives instead of duplicating storage/sync logic inside hook bodies.
- Session synchronization policy remains unchanged while becoming easier to test in isolation.

### Verification
- IDE lint diagnostics report no issues in changed frontend files.
- Existing hook signatures and return shapes were preserved.

### Files Changed
- `client/src/lib/session/session-sync.ts`
- `client/src/hooks/use-session.ts`
- `client/src/hooks/use-filter-state.ts`
