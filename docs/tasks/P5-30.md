# P5-30: Interactive viewer fallback to rendered event visibility source

Date: 2026-03-26
Status: DONE

## Goal

Prevent false `No curves visible` states in Interactive view when Grid still has rendered/cached curves.

## Problem

`InteractiveViewer` previously derived visibility only from `allSelectedEventIds`. After edit-route navigation, selection could be empty while `rendered_event_ids` and cached plot data still existed, producing an empty-state mismatch between Interactive and Grid.

## Implementation

- Added `effectiveEventIds` in `InteractiveViewer`:
  - Use `allSelectedEventIds` when non-empty.
  - Fall back to `renderedEventIds` when selection is empty.
- Updated `visibleEventIds` computation to filter `effectiveEventIds` by `curveVisibility`.
- Kept existing state priority unchanged: loading -> error -> no visible curves -> plot render.
- Kept tab persistence behavior unchanged.

## Verification

- Confirmed empty-state gate now depends on effective visibility source.
- Verified no lint errors in `InteractiveViewer.tsx`.
