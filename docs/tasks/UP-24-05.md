# UP-24-05 — Read persisted damage states in Inspect Damage

Date: 2026-06-11
Status: DONE

## Summary

Inspect Damage now reads persisted `event_channel_damage` rows instead of computing damage on read. The page shows current and stale values with warnings/badges, an empty state with Calculate Damage when no successful result exists, prerequisite guidance when derived data is missing, and failure-report context from recent tasks.

## Implementation

### Server

- Added `server/services/damage_inspect.py` to build inspect responses from persisted rows and per-scope state (prerequisites, failure reports, active tasks).
- Extended `DamageInspectResponse` with `has_stale_values` and `scopes`; extended `DamageCell` with `base_damage` and `stale_reason`.
- Replaced compute-on-read logic in `POST /damage/inspect` with persisted reads only.
- Added `POST /damage/calculate` (write users) to start or reuse `damage_calculation` tasks for a program/version scope.
- Added `list_event_channel_damage_for_event_ids` and `find_latest_failed_damage_calculation_task` repository helpers.

### Client

- Added `client/src/features/inspect-damage/` view-state helpers and calculate-response wiring.
- Inspect Damage page auto-fetches persisted rows on selection via React Query; refetches while damage calculation runs.
- UI: stale page banner, column/cell stale badges, empty state with Calculate Damage, prerequisite guidance, failure summaries, read-only action gating, damage progress modal reuse.
