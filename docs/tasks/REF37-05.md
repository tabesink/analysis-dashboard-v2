## REF37-05 - Derived Lane Command/Query Boundaries

Date: 2026-06-16

### Behavior Changed

- Added explicit route behavior coverage that channel-map command endpoints remain channel-reprocess scoped:
  - YAML upload route returns a `channel_reprocess` task response.
  - Manual channel-map save route returns a `channel_reprocess` task response.
- Added explicit route behavior coverage that schedule command endpoints start damage calculation only when prerequisites are ready:
  - Schedule attach returns `validation_blocked` with no damage task when prerequisites are missing.
  - Schedule attach returns `calculation_started` when prerequisites are current.
  - Schedule save returns a damage task response (`calculation_started` or `reused_active_task`) when prerequisites are current.
- Added explicit boundary coverage that Inspect Damage is query-only while backfill is the write-user repair command:
  - Inspect does not create damage-calculation tasks for missing persisted rows.
  - Backfill creates/reuses a `damage_calculation` task for the same scope.

### Interfaces Changed

- No API shape changes were required.
- Existing response contracts remain unchanged:
  - Channel-map command responses stay `DerivedTaskStartResponse` (`task_kind = channel_reprocess`).
  - Schedule command responses stay `DurabilitySchedule*Response` with `schedule_command_outcome` and optional damage task fields.
  - Inspect/backfill endpoints keep their existing request/response models.
