# P11-11: Initial `UnifiedStore` repository extraction (users/sessions first)

## Summary

Extracted low-risk `users` and `sessions` storage operations behind repository
modules while keeping `UnifiedStore` as the sole DuckDB connection owner.

## Changes

- Added `server/storage/repositories/users_repository.py` with user CRUD and
  user-account utility queries formerly inline in `UnifiedStore`.
- Added `server/storage/repositories/sessions_repository.py` with session
  get/upsert/delete operations formerly inline in `UnifiedStore`.
- Added repository package entrypoint:
  `server/storage/repositories/__init__.py`.
- Updated `server/storage/database.py`:
  - instantiate repositories in `UnifiedStore.__init__`
  - keep public `UnifiedStore` user/session methods and delegate to repositories
  - preserve existing API shape for services/routers/tests
  - keep `UnifiedStore` as single connection owner (repositories consume
    `read_connection`/`write_connection` from the store)
- Added focused storage coverage:
  `test_unified_store_session_operations_via_repository_extraction`.
