# DB14-01: Source artifact storage service and ledger records

**Status:** DONE (2026-06-09)

## Summary

Introduced `SourceArtifactStorageService` to write immutable original CSV/RSP upload bytes under `data/artifacts/sources/` and record lineage in the new `source_artifacts` DuckDB table.

## Changes

- `server/services/source_artifact_storage.py` — portable `artifact://` URI build/resolve, SHA-256 checksums, path safety validation.
- `server/schema.yaml` — `source_artifacts` table + `seq_source_artifact_id`.
- `server/storage/database.py` — upsert/list helpers; scope delete clears ledger rows and filesystem bytes; portability excludes source artifacts (like `ingestion_artifacts`).
- `server/services/ingestion.py` — upload ingestion stores source artifacts before parse/validate.
- `docs/decisions/log.md` — DEC-073 artifact URI contract.

## Tests

- `tests/server/services/test_source_artifact_storage.py` — service behavior (CSV, RSP, unsafe URI rejection).
- `tests/server/services/test_ingestion_service_status.py` — ingest integration + scope delete cleanup.

## Deferred to DB14-02

- Canonical CSV artifact derivation as a separate artifact type.
- `ingestion_run` and `event_source_link` lineage tables.
