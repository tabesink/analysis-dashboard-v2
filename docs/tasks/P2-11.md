# P2-11: CSV upload SSE progress with per-event commit milestones

Date: 2026-03-30
Status: DONE

## Goal

Expose real-time progress during CSV ingestion after upload bytes complete, with progress increments tied to committed event writes (including LTTB), while keeping the architecture lightweight and multi-user aware.

## Implementation

- Replaced legacy CSV upload contract with:
  - `POST /api/v1/upload/folder/start` (multipart upload) -> returns `task_id`
  - `GET /api/v1/upload/folder/events/{task_id}` (SSE stream)
- Added upload task models in backend and client:
  - `UploadTaskStartResponse`
  - `UploadTaskEvent`
- Added DuckDB-backed `upload_tasks` table with short-lived task retention and cleanup support.
- Added `UnifiedStore` task methods:
  - create/update/get upload tasks
  - delete expired tasks
- Refactored ingestion write phase to per-event commits and introduced post-commit callback updates for progress.
- Updated upload hook to:
  - use XHR progress for 0-10%
  - subscribe to SSE for 10-100%
  - update message with per-event progress text

## Multi-User Behavior

- Task rows are stored with `created_by_user_id`.
- SSE endpoint scopes access by creator user id (creator-only stream visibility).
- Shared dataset visibility remains unchanged (authenticated users can still view shared upload data as before).
