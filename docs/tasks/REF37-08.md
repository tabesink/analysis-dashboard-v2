# REF37-08 - Clarify database export and connection lane

Date: 2026-06-16

## Behavior changed

- Database administration routes for listing, creating, connecting, deleting, and exporting databases are now consistently admin-only.
- Database settings UI copy now clearly frames this area as a whole-database administration lane, separate from folder upload.
- Export lifecycle feedback now keeps detailed progress/cancel/download readiness in the modal while adding lightweight toasts for start, cancellation, success, and failure.

## Interfaces changed

- `server/routers/export.py`
  - `GET /api/v1/export/database/list` now enforces admin access.
  - `POST /api/v1/export/database/connect` now enforces admin access.
- `client/src/hooks/use-database-switch.ts`
  - connect action now requires admin and reports admin-required feedback.
  - `UseDatabaseSwitchOptions` no longer accepts `canWrite`; it is admin-only scoped.
- `client/src/components/settings/panels/DatabaseSettingsPanel.tsx`
  - updated `useDatabaseSwitch` usage to admin-only options.
- `client/src/components/upload/DatabaseSection.tsx`
  - updated labels/copy to distinguish database administration from folder upload.
- `client/src/hooks/use-database-operation.ts`
  - export flow now emits lightweight toasts for start/cancel/failure while preserving modal-owned long-running status.

## Tests added or updated

- Updated `tests/server/routers/test_export_router.py`:
  - writer/reader users are forbidden from database admin lane routes (list, export-start, create, connect, delete).
  - replaced prior writer-connect allowance with explicit non-admin rejection assertions.
- Updated `client/src/components/upload/DatabaseSection.test.tsx`:
  - verifies admin lane labels (`Database Administration`, `Create Managed Database`, `Switch Active Database`, `Export Active Database`) and confirms no import control.

## Follow-on assumptions for next slice

- Database administration now has explicit admin-only route guards and matching UI wording, so later slices can treat it as a separate lane from folder upload.
- Export progress/download modal contract remains intact for downstream task-kind reconciliation work.

## Decisions intentionally unchanged

- Folder upload lane policy, progress sequencing, and cancellation behavior are unchanged.
- Export task polling/download mechanics remain unchanged.
