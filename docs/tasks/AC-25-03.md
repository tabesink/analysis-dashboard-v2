# AC-25-03 — Database-page banner for background channel reprocess

**Status:** DONE (2026-06-11)

## Summary

When channel reprocess runs in the background (modal dismissed, Edit Metadata closed), the Database page shows a compact scoped banner with program/version label, live progress message, and **Reopen progress** action. Reuses existing channel reprocess store state; no global task center.

## Changes

- Added `selectDatabaseChannelReprocessBanners` — pure visibility selector (running + modal dismissed + editor not open for same scope).
- Added `DatabaseChannelReprocessBanner` and `DatabaseChannelReprocessBanners` container; mounted at top of Database page shell.
- Inline Edit Metadata banner unchanged; database-page banner suppressed when editor is open for the same scope.

## Tests

- `database-channel-reprocess-banner.test.ts` — visibility rules, completion suppression, fallback running label.
- `DatabaseChannelReprocessBanners.test.tsx` — scoped markup, reopen affordance, editor-open suppression, read-only-safe content (no save/upload actions).

## Acceptance criteria

- [x] Banner appears when reprocess running, modal dismissed, editor closed for scope.
- [x] Banner shows program id and version.
- [x] Banner shows live progress message or short running label.
- [x] Reopen progress opens shell-mounted modal (`reopenChannelReprocessModal`).
- [x] Banner hidden after summary acknowledged (store entry cleared).
- [x] No duplicate with Edit Metadata inline banner for same scope.
- [x] Read-only users see running state without write actions on banner.
- [x] Client tests cover visibility, reopen, editor suppression.
