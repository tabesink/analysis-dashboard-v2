# P5-20: UploadDataSection component

Date: 2026-03-30
Status: DONE

## Goal

Enforce upload `Status` behavior by role: default `Pending` for all users, editable for admins, and locked/ghosted for non-admin users.

## Implementation

- Updated upload form state defaults and reset paths in `client/src/app/database/page.tsx` so `Status` starts as `Pending` and resets to `Pending`.
- Adjusted active-filter detection to treat default `Status=Pending` as baseline (not an active custom filter).
- Updated `client/src/components/upload/UploadDataSection.tsx` to always render `Status`.
- Added role-based lock behavior for `Status`: non-admin users see a disabled/muted select and cannot trigger `onValueChange`.
- Kept admin-only payload override behavior unchanged (client sends `status` override only for admins; backend keeps non-admin effective status as `Pending`).
- Added ingestion service regression tests in `tests/server/services/test_ingestion_service_status.py` covering both non-admin and admin paths.

## Verification

- Confirmed code paths preserve backend enforcement: `effective_status = status_value if is_admin else "Pending"` in `server/services/ingestion.py`.
- Added tests to verify:
  - non-admin requests with `status_value="Approved"` persist `Pending`,
  - admin requests with `status_value="Obsolete"` persist `Obsolete`.
- Lint/tests should be run for the touched files in CI/local validation.
