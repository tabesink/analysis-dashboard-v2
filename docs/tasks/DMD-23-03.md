# DMD-23-03 — Harden schedule dirty state, permissions, refresh, and accessibility

Completed: 2026-06-10

## Summary

Extended the metadata popup dialog from two-section to three-section dirty-close handling. `isMetadataDialogDirty` now combines metadata, channel-map, and schedule dirty flags for close, Escape, outside-click, and pending scope-change prompts. Confirming discard on scope change clears all three section dirty flags. Section switching remains unprompted with mounted panels retaining drafts. Schedule save/upload already invalidate `program-version-schedule` and hydrate clean baselines via existing helpers from DMD-23-01.

## Manual verification

- Edit schedule rows in popup, close → discard prompt; Keep editing preserves draft.
- Edit schedule, click pencil on another row → scope-change discard prompt.
- Switch Edit Metadata ↔ Assign Channels ↔ Durability Schedule with unsaved schedule edits → no prompt; drafts persist.
- Read-only user: no upload in empty schedule state; table read-only; no Reset/Save.
- Save or upload schedule → panel shows saved baseline without stale dirty state.
