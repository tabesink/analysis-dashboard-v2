# P8-16 Implementation Notes

Date: 2026-03-09
Status: DONE

## Scope

- Replace legacy `phase` with boolean applicability fields: `rfq`, `dv`, `pv`, `post_prod`.
- Apply replacement across schema, API, filter behavior, and UI.
- Make Edit Metadata visibly reflect the new model:
  - numeric raw inputs for GVW/FGAWR/RGAWR
  - checkbox-based applicable phases in right-hand section.

## Changes Made

### 1) Backend schema + models

- Updated `server/schema.yaml`:
  - removed `phase` filter column config
  - added boolean columns with filter config:
    - `rfq`, `dv`, `pv`, `post_prod`
  - shifted filter orders accordingly
- Updated `server/storage/database.py`:
  - added compatibility `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` for new booleans
  - defaulted null boolean values to `FALSE` for existing rows
- Updated Pydantic models:
  - `server/models/dashboard.py`
  - `server/models/upload.py`

### 2) Backend filter/query behavior

- Added `server/utils/boolean_filters.py` to map UI values:
  - `Applicable` -> `TRUE`
  - `Not Applicable` -> `FALSE`
- Wired boolean filter helper into:
  - `server/services/query.py` (`get_events`)
  - `server/storage/database.py` (`get_program_ids`, `get_versions`)
- Updated dashboard routes:
  - replaced `phase` query/filter plumbing with `rfq/dv/pv/post_prod`
  - returned new boolean metadata fields in event payloads
  - added boolean-safe metadata normalization in update endpoints

### 3) Upload and dataset metadata flow

- Updated `server/routers/upload.py`:
  - accepts `rfq/dv/pv/post_prod` form fields
  - stores booleans in event metadata
  - dataset listing now selects boolean columns instead of `phase`
- Updated client upload contracts:
  - `client/src/types/upload.ts`
  - `client/src/lib/api/upload.ts` (boolean form-data serialization)

### 4) Frontend visible UI updates

- Updated `client/src/app/database/edit/page.tsx`:
  - removed legacy phase field from metadata option rendering
  - added numeric raw inputs for:
    - Gross Vehicle Weight (lbs)
    - FGAWR (lbs)
    - RGAWR (lbs)
  - added right-side applicable-phase checkbox block:
    - RFQ, DV, PV, Post-Prod
  - save/reset/prefill logic extended for weight + phase-boolean drafts

### 5) Global filter and color-category alignment

- Regenerated `client/src/config/filters.ts` from updated schema.
- Updated phase references in:
  - `client/src/components/dashboard/side-panel/global-filters/constants.ts`
  - `client/src/components/dashboard/side-panel/global-filters/utils.ts` (boolean display conversion)
  - `client/src/hooks/use-event-catalog.ts` (boolean matching conversion)
  - `client/src/components/dashboard/color-legend/ColorLegend.tsx`
  - `client/src/stores/color-selection-store.ts`

### 6) Database page compatibility updates

- Updated `client/src/app/database/page.tsx` to:
  - support boolean metadata rendering (`Applicable`/`Not Applicable`)
  - remove legacy upload mapping for `Phase`
