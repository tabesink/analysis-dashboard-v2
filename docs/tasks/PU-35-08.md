# PU-35-08 — Add absolute-by-event heatmap card

## Behavior added

- Extended `buildDamage2DPlotSpec` to support `absolute_by_event` with a `heatmap` chart spec path.
- Added event/channel filtering that respects selected channels plus selected Reference and Target event ids.
- Added shared value-mode + damage-scale behavior (`absolute`/`normalized`, `linear`/`log`) for heatmap cells using `applyDamageScale`.
- Added heatmap truncation safeguards (max rendered cell budget) that emit a readable warning and structured density summary instead of overfilling card layout.
- Added `AbsoluteByEventHeatmapCard` renderer with event/channel matrix labels and native hover tooltip content showing event, program/version, channel, dataset, and value.
- Wired `Damage2DPlotCard` and `DamagePlotView` so `absolute_by_event` uses the new heatmap renderer in the 2×2 card grid while preserving focused 3D routing.

## Interfaces changed

- `buildDamage2DPlotSpec` input now accepts optional:
  - `referenceEventIds?: readonly string[]`
  - `targetEventIds?: readonly string[]`
- `Damage2DPlotSpec` now optionally includes:
  - `heatmapCells`
  - `heatmapSummary`
- New component: `AbsoluteByEventHeatmapCard`.
