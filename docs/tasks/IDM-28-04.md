# IDM-28-04 — Channel reprocess uses per-event headers

**Status:** DONE (2026-06-11)

## Summary

Hardened channel reprocess LTTB regeneration to resolve plot axis channel names per event via the shared header provider and resolver, aligned with damage lookup (IDM-28-03). Added mixed-fixture regression tests and explicit artifact failure when header resolution or plot extraction produces no cross-plot data.

## Changes

- `server/services/ingestion.py` — `_plot_lookup_channels_for_event()` now returns resolver errors for index-based maps; `_process_retained_artifacts()` fails the artifact when lookup fails or no LTTB plots are extracted.
- `tests/server/services/test_channel_reprocess_per_event_headers.py` — mixed Moog + abbreviated fixture batch, first-artifact ordering independence, empty extraction failure semantics.

## Behavior

1. For each plot on each event, load headers via `EventHeaderProvider` and resolve `x_channel` / `y_channel` from column indices when the persisted map uses `col_N`.
2. Resolver errors (`missing_headers`, `column_out_of_range`) fail the artifact with the plot key prefixed to the message.
3. When all plots extract empty series after resolution, fail with a clear no-cross-plot-data message (consistent with existing missing-raw failure semantics).
4. Legacy explicit title strings in the channel map continue to bypass per-event resolution.
