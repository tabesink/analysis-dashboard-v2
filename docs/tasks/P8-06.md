# P8-06: Consistent cache invalidation on all write paths

## Summary

Audited write endpoints in `upload` and `dashboard` and filled the remaining
cache invalidation gap for custom-field mutation paths.

## Changes

- Added `QueryService.invalidate_filter_option_caches()` in
  `server/services/query.py`.
- Updated dashboard custom-field write routes to invalidate filter-options cache:
  - `POST /api/v1/dashboard/custom-fields`
  - `PUT /api/v1/dashboard/custom-fields/{field_key}/program-values/{program_id}`
- Verified existing write-path cache behavior remained in place:
  - delete/purge/program-scope delete paths in `upload` invalidate event-related
    cache groups via `query_service.invalidate_event_caches()`
  - metadata update paths in `QueryService` invalidate event/program/version
    cache groups
  - filter-options update/reset already invalidated filter-options cache
- Added focused regression test:
  - `test_invalidate_filter_option_caches_clears_filter_prefix`
