# UP-24-03 — Persist latest schedule-driven load-history damage

Date: 2026-06-11
Status: DONE

## Summary

Schedule upload and save now trigger background `damage_calculation` derived-data tasks when raw load histories and cross-plot data are current. Missing or stale prerequisites return a structured report without creating a task row. Latest damage results persist in `event_channel_damage`.

## Implementation

- Added `event_channel_damage` table (composite PK on `event_id`, `channel_key`) with current/stale/error status, schedule lineage, and scheduled damage fields.
- Modules: `schedule_event_rows`, `schedule_damage_validation`, `schedule_damage_prerequisites`, `schedule_damage_calculation`, `damage_calculation_task`, `damage_calculation_progress`.
- Schedule attach hydrates and persists `event_rows` before damage trigger; schedule save/upload responses include `damage_task_id` or `damage_prerequisite_report`.
- Channel reprocess and schedule changes mark prior `current` damage rows stale (`channel_reprocess_required`, `schedule_changed`).
- Validation failures inside the damage task return transient `failure_report` JSON; stale rows are preserved.
