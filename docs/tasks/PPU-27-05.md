# PPU-27-05 — Refresh derived views and surface automatic precompute feedback

**Status:** DONE (2026-06-11)

## Summary

Centralized client-side derived-task completion handling so automatic and manual `damage_calculation` work refreshes dependent views consistently and surfaces blocked/failed automatic precompute with concise toast-only feedback.

## Implementation

- `client/src/lib/damage-calculation-cache.ts`
  - `invalidateQueriesAfterDamageCalculation()` refreshes `damage-inspect` and scoped `program-version-schedule` queries.
- `client/src/features/edit-metadata/lib/damage-calculation-task-completion.ts`
  - `resolveDamageCalculationCompletionBehavior()` distinguishes manual modal summaries from automatic toast-only failures and background completion.
- `client/src/features/edit-metadata/lib/blocked-precompute-feedback.ts`
  - Shared blocked-precompute toast copy with program/version scope and high-level prerequisite naming.
- `client/src/features/edit-metadata/lib/channel-reprocess-follow-up.ts`
  - Applies server `precompute_follow_up` after channel reprocess completion to auto-track damage tasks or emit blocked toasts.
- `client/src/stores/damage-calculation-store.ts`
  - Uses centralized invalidation/completion behavior; supports `origin` and `openModal` on `trackDamageCalculationTask`.
- `server/services/ingestion.py`, `server/services/post_upload_precompute.py`
  - Channel reprocess task results now include `precompute_follow_up` for client follow-up wiring.
