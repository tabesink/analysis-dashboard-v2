# PU-35-12 - Database-owned damage table access

## Behavior added or changed

- `/database` now has central tabs: `Datasets` and `Damage Table`.
- `Damage Table` now loads all non-deleted events that already have persisted `event_channel_damage` rows; users do not need to preselect scopes in Inspect Damage.
- Damage-table versions are collapsed by default so event rows under each version remain hidden until expanded.
- `/inspect-damage` no longer renders the old table tab path and is now plot-focused.

## Interfaces changed

- Client API: `damageApi.inspectCalculated()` posts to `/api/v1/damage/inspect` with `include_all_calculated: true`.
- Server request model: `DamageInspectRequest` adds `include_all_calculated: bool = False`.
- Server inspect service: `build_damage_inspect_response(..., include_all_calculated: bool = False)` can resolve event IDs from persisted damage rows.
- Database storage helper: `list_event_ids_with_persisted_damage()` returns non-deleted event IDs with damage rows in stable program/version/event order.

## Tests added

- `tests/server/services/test_damage_inspect.py`
  - Added coverage for `include_all_calculated=True` to prove deleted events are excluded and row ordering is stable.
- `tests/server/routers/test_damage_router.py`
  - Added API coverage proving `include_all_calculated` returns all calculated rows even when explicit `event_ids` are irrelevant.
- `client/src/features/inspect-damage/components/DamageTableView.test.tsx`
  - Added coverage proving version groups render collapsed by default.

## Follow-on assumptions

- Database damage-table preferences are route-local (`database_damage_table_prefs_v1`) and separate from dataset table preferences.
- Existing repo-wide `eslint` failures outside this slice remain pre-existing and unchanged.

## Decisions intentionally unchanged

- Persisted damage read path remains query-only and does not trigger compute-on-read.
- Existing Database upload/filter side panel behavior is unchanged.
