# UP-24-01 — Add lean async channel reprocess tasks

Date: 2026-06-11
Status: DONE

## Summary

Channel-map save and YAML upload now start a background `channel_reprocess` derived-data task instead of blocking until retained-artifact reprocessing finishes. Task rows extend the existing `upload_tasks` table with lean progress fields.

## Implementation

- Extended `upload_tasks` with additive columns: `task_kind`, `sub_phase`, `progress_message`, `scope_json`.
- Added `DerivedTaskStartResponse` and `DerivedTaskStatusEvent` models.
- Extracted `_persist_channel_map_only` and `_process_retained_artifacts` from synchronous channel-map processing; async path runs reprocessing in a daemon thread with progress callbacks that update task rows outside artifact write transactions.
- Added `start_channel_reprocess_from_save` / `start_channel_reprocess_from_yaml` on `IngestionService`.
- Dashboard routes:
  - `PUT /program-version/channel-map` → returns task start payload
  - `POST /program-version/channel-map/upload` → same
  - `GET /derived-data/task/{task_id}` → creator-scoped lean poll payload
- Active derived-data task reuse per program/version via `find_active_derived_data_task`.
- Locked progress message helpers in `server/services/channel_reprocess_progress.py`.
