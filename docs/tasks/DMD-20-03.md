# DMD-20-03 — Harden dialog save, permissions, dirty-close, and refresh behavior

Completed: 2026-06-10

## Summary

Made the Database table metadata edit dialog production-ready: save still flows through `EditMetadataPanel` → `updateProgramVersionMetadata` → `invalidateQueriesAfterMetadataSave`; write/admin permission parity matches the full-page route; dirty metadata cannot be silently discarded on close or scope switch; Radix dialog close/Escape/overlay requests are intercepted when dirty.

| File | Role |

## Manual verification

- Open dialog from version row A, edit a field, close → discard prompt appears; Keep editing preserves draft; Discard closes.
- Save metadata from dialog → database table status/rollup updates without page refresh (via `datasets` invalidation).
- Click pencil on version row B while dialog open with clean state → scope switches immediately.
- Click pencil on version row B while dirty → discard prompt offers switch or cancel.
- Non-admin: Status field disabled with admin guidance; other fields editable.
- Keyboard: Tab reaches close control, nav item, panel fields, Save; Escape triggers close/discard flow.
