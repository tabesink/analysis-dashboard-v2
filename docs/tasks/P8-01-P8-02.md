# P8-01 / P8-02 Implementation Notes

Date: 2026-03-09
Status: DONE

## Scope

- P8-01: Fix runtime kwarg mismatch on upload ingestion call.
- P8-02: Enforce owner/admin authorization for event metadata updates.

## Changes Made

### 1) Upload kwarg alignment (`P8-01`)

- File: `server/routers/upload.py`
- Updated `upload_folder()` call to ingestion service from:
  - `status=status_value or "Pending"`
- To:
  - `status_value=status_value or "Pending"`

This now matches `IngestionService.ingest(..., status_value: str = "Pending", ...)` in `server/services/ingestion.py` and removes the upload-time runtime mismatch.

### 2) Metadata ownership enforcement (`P8-02`)

- File: `server/routers/dashboard.py`
- Updated `update_event_metadata()` to enforce:
  - admin can update any event metadata
  - non-admin can update only when `existing.uploaded_by_user_id == current_user.id`
  - otherwise returns `403` with ownership error detail

The existing non-admin status guard remains unchanged:
- non-admin users still cannot update `status` even when they own the event.
