# DB14-07: Transfer package export/import with artifact validation

**Status:** DONE (2026-06-09)

## Summary

Admin database export now produces a transfer package that includes lineage Parquet tables, portable artifact bytes, and a `manifest.json` with SHA-256 checksums. Import validates package integrity before replacing load data and installs artifacts into the runtime data root.

## Public interface

- `TransferPackageService.export_package(export_dir)` — export tables + artifacts + manifest
- `TransferPackageService.validate_package(package_root)` — non-destructive validation
- `TransferPackageService.import_package(package_root)` — validate, import tables, install artifacts
- `ExportService` admin export/import auto-detects transfer packages via `manifest.json`

## Legacy path

`UnifiedStore.export_to_parquet()` and `import_from_parquet()` without manifest continue to use `LOAD_DATA_PORTABILITY_TABLES` only.

## Tests

`tests/server/services/test_transfer_package.py` — manifest/artifact export, checksum and missing-artifact rejection, round-trip, legacy export unchanged.

`tests/server/services/test_export_service.py` — export task produces transfer package; ZIP validation accepts transfer archives.
