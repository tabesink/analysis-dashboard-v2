# P11-08: Declared-vs-live schema doctor classification report

## Summary

Added the deferred schema doctor slice on top of the DEC-049 DDL ownership refactor. Migration diff now classifies declared-vs-live table state as `OK`, `MISSING`, `TYPE_MISMATCH`, or `DRIFT`.

## Changes

- Extended `MigrationRunner.generate_migration_diff()` to:
  - read live table/column metadata from DuckDB `information_schema`,
  - compare it to declared schema tables/columns from `SchemaLoader`,
  - classify each declared table (plus extra live tables) with doctor statuses.
- Added type normalization in migration diff to avoid false mismatches from DuckDB alias/format differences.
- Preserved compatibility fields already used by existing scripts:
  - `tables_in_schema`
  - `tables_in_db`
  - `missing_tables`
  - `extra_tables`
- Added new report fields for diagnosis:
  - `doctor_report` (table entries with status and details)
  - `doctor_summary` (status counts)
- Added behavior test coverage proving all four statuses:
  - `test_schema_doctor_classifies_ok_missing_type_mismatch_and_drift`
