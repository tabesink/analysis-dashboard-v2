# SPV-33-02 - Parameterize Load Data event tree sections

Date: 2026-06-12
Status: DONE

## Behavior added or changed

- Refactored `client/src/components/dashboard/side-panel/LoadDataSection.tsx` so shared load-data tree behavior now lives in `LoadDataEventTreeSection`, parameterized by section title, selected IDs, update callback, and empty/subtitle guidance.
- Preserved existing single-pool dashboard behavior by keeping `LoadDataSection` as a wrapper around the new primitive with the same `selected_event_ids` read/write behavior and legacy `program_ids`/`versions` clear-on-none behavior.
- Added `client/src/components/dashboard/side-panel/ComparisonLoadDataSections.tsx` to render `Reference Load Data` and `Target Load Data` from one filtered event catalog while routing updates independently to either `comparison.reference.selected_event_ids` or `comparison.target.selected_event_ids`.
- Added comparison empty-guidance resolution for filter-empty, non-selectable, and missing-reference/target-selection states.

## Interfaces changed

- Added exported component: `LoadDataEventTreeSection`.
- Added exported component: `ComparisonLoadDataSections`.
- Added exported helper: `resolveComparisonLoadDataEmptyMessage()`.
- Updated side-panel barrel export in `client/src/components/dashboard/side-panel/index.ts` to include `ComparisonLoadDataSections`.

## Tests added and what they prove

- `client/src/components/dashboard/side-panel/LoadDataSection.test.tsx`
  - proves single-pool toggle/select-all/select-none behavior remains unchanged,
  - proves default empty-state message wiring remains intact.
- `client/src/components/dashboard/side-panel/ComparisonLoadDataSections.test.tsx`
  - proves distinct Reference/Target section labeling and subtitle intent,
  - proves reference writes only patch `comparison.reference.selected_event_ids`,
  - proves target writes only patch `comparison.target.selected_event_ids`,
  - proves filter-empty guidance messaging is emitted for both sections.

## Follow-on assumptions for next issue

- Downstream comparison UI slices can mount `ComparisonLoadDataSections` directly against the shared filtered catalog and comparison-session merge path.
- Channel and plot slices can assume Reference and Target event selections are independently writable without changing existing single-pool load-data routes.

## Decisions intentionally left unchanged

- Channel selection semantics are unchanged.
- Plot fact construction and aggregation semantics are unchanged.
- Global filter ownership remains shared and outside this slice.
- Existing dashboard and inspect-damage single-pool event-tree behavior remains intact.
