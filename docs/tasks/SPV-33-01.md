# SPV-33-01 - Harden comparison state helpers and persistence

Date: 2026-06-12
Status: DONE

## Behavior added or changed

- Added dedicated comparison-state helper tests in `client/src/lib/damage-comparison-state.test.ts` covering:
  - canonical default state,
  - partial merge semantics,
  - deterministic event-id and channel-key pruning,
  - program/version derivation from selected comparison event IDs.
- Added `deriveProgramVersionScopeFromComparisonState()` in `client/src/lib/damage-comparison-state.ts`.
- Hardened session backup/default merge behavior in `client/src/lib/session/session-sync.ts` so `inspect_damage_state.comparison` is defaulted, persisted in backup payloads, and merged on update.
- Extended `client/src/hooks/use-inspect-damage-state.test.ts` regression coverage for comparison defaults and merge behavior in `mergeInspectDamageState()` and `mergeSessionUpdates()`.

## Interfaces changed

- Added exported helper:
  - `deriveProgramVersionScopeFromComparisonState(state, eventCatalog)`
- No shape changes to `DamageComparisonState`.

## Tests added and what they prove

- `client/src/lib/damage-comparison-state.test.ts`
  - proves default/merge/prune invariants,
  - proves derived program/version scope is deterministic and event-driven.
- `client/src/hooks/use-inspect-damage-state.test.ts`
  - proves inspect-damage session defaults include comparison state,
  - proves backup restore keeps comparison state,
  - proves table-preference updates do not drop comparison selections.

## Follow-on assumptions for next issue

- Downstream slices can rely on comparison state persistence across refresh/navigation through session backup/default merge paths.
- Event-tree and raw-fact slices can use `deriveProgramVersionScopeFromComparisonState()` as the canonical adapter for program/version derivation from selected events.

## Decisions intentionally left unchanged

- `DamageComparisonState` contract remains unchanged.
- Single-pool dashboard `data_state.selected_event_ids` remains independent from comparison route state.
- No backend API or plot aggregation behavior was changed in this slice.
