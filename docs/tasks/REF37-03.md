# REF37-03 - Client upload policy and metadata mapping

## Behavior changed

- Added a shared client upload policy helper (`client/src/features/database-upload/upload-policy.ts`) that classifies selected files consistently for both side-panel display and submit preflight.
- Upload preflight toast behavior is now centralized: missing metadata, no uploadable data files, mixed CSV/RSP selection rejection, and ignored unrelated file notices all run through one helper-driven path.
- Existing user-facing labels remain unchanged, including required-field prompts and the "Program ID" form field continuing to map to the `job_number` payload key.

## Interfaces changed

- `DatabasePage` now consumes shared helpers for:
  - selected-file summary (`summarizeUploadSelection`)
  - required metadata validation and payload mapping (`buildUploadMetadataPayload`)
  - lightweight toast feedback (`notifyUploadPreflightFeedback`)
- `UploadDataSection` now consumes the same selected-file summary helper so disabled/error state rules match submit-time behavior.
- No API endpoints, route shapes, or server contracts changed.

## Tests added

- Added `client/src/features/database-upload/upload-policy.test.ts`:
  - Covers selected-file classification for CSV-only, RSP-only, mixed CSV/RSP rejection, optional channel-map companion handling, and unsupported/ignored files.
  - Covers required metadata validation and mapping, including `Program ID` form label mapping to `job_number`.
  - Covers toast feedback behavior for mixed-file rejection and ignored-file informational messaging.

## Follow-on assumptions for next slice

- Client upload display and submit preflight logic now share one policy source of truth, reducing drift risk as progress/cancellation behavior evolves.
- Future slices can extend upload modal/progress flows without re-implementing file classification or metadata mapping.

## Decisions intentionally unchanged

- Server upload route behavior and policy enforcement.
- Upload progress rendering and operation modal workflow.
- Existing database-side upload layout/component structure (no broad tree movement).
