# Triage

Move issues through a small state machine of triage roles using the tracker described in `docs/agents/issue-tracker.md` and labels from `docs/agents/triage-labels.md`.

Every comment or issue posted during triage must start with:

```markdown
> *This was generated by AI during triage.*
```

## Reference Docs

- `AGENT-BRIEF.md`: durable agent brief format.
- `OUT-OF-SCOPE.md`: how to record rejected or out-of-scope requests.

## Roles

Two category roles:

- `bug`: something is broken.
- `enhancement`: a new feature or improvement.

Five state roles:

- `needs-triage`: maintainer needs to evaluate.
- `needs-info`: waiting on reporter.
- `ready-for-agent`: fully specified and agent-ready.
- `ready-for-human`: needs human implementation.
- `wontfix`: will not be actioned.

Every triaged issue should carry exactly one category role and one state role. If state roles conflict, flag it and ask before changing anything.

## Show What Needs Attention

Query the issue tracker and present three buckets, oldest first:

1. Unlabeled issues.
2. `needs-triage` issues.
3. `needs-info` issues with reporter activity since the last triage note.

Show counts and one-line summaries, then let the maintainer pick.

## Triage A Specific Issue

1. Gather context: read the full issue body, comments, labels, reporter, dates, and prior triage notes.
2. Explore the relevant code using domain docs and decisions.
3. Recommend category and state with reasoning.
4. For bugs, attempt reproduction before grilling.
5. If the issue needs clarification, use `grill-with-docs`.
6. Apply the outcome only after confirming the intended tracker changes.

## Outcomes

- `ready-for-agent`: post an agent brief using `AGENT-BRIEF.md`.
- `ready-for-human`: post the same structure, explaining why it cannot be delegated.
- `needs-info`: post concise triage notes and specific questions.
- `wontfix`: post a polite explanation, optionally record recurring rationale using `OUT-OF-SCOPE.md`, then close if appropriate.
- `needs-triage`: apply the role; optionally comment if partial progress exists.

## Needs-Info Template

```markdown
## Triage Notes

**What we've established so far:**

- point 1
- point 2

**What we still need from you (@reporter):**

- question 1
- question 2
```

Do not re-ask resolved questions when resuming a previous triage session.
